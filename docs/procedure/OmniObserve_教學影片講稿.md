# 線上會議輔助系統教學影片講稿

本文件提供線上會議輔助系統教學影片的主要參考內容，實際使用方式可依研究設計、系統版本與現場需求調整。內容包含任務前教學影片與最後說明影片：

1. `A_common_system_intro`：共通系統介紹。
2. `B_lost_at_sea_no_cue`：Lost-at-Sea，no-cue 版本。
3. `B_lost_at_sea_with_cue`：Lost-at-Sea，with-cue 版本。
4. `C_enhance_the_poster_no_cue`：Enhance-the-Poster，no-cue 版本。
5. `C_enhance_the_poster_with_cue`：Enhance-the-Poster，with-cue 版本。
6. `D_final_debrief`：最後說明，應在需要保密的研究程序與相關訪談完成後播放。

錄影時不要說「實驗組 / 對照組」。面向參與者時，用「這一輪不會顯示系統提示」和「這一輪可能會顯示系統提示」即可。

## 0. 現場播放規則與 counterbalance 矩陣

- 每場一開始播放共通系統介紹 `A_common_system_intro`。
- 每輪任務開始前，只播放該任務、該 condition 對應的一支任務影片。
- 不要在同一輪任務前同時播放 no-cue 和 with-cue 影片。
- 任務影片一律說「這一輪任務」，不要說「第一個任務」或「第二個任務」。
- 若某任務是第二輪，不需要重播共通系統介紹，只播放該任務影片即可。
- 任務前影片中不要提暗樁、研究假設、後台判斷規則或 Lost-at-Sea 專家答案。
- 最後說明影片應在需要保密的研究程序與相關訪談完成後播放。

| 任務 | Condition | 播放影片 |
| --- | --- | --- |
| Lost-at-Sea | no-cue | A + B lost no-cue |
| Lost-at-Sea | with-cue | A + B lost with-cue |
| Enhance-the-Poster | no-cue | A + C poster no-cue |
| Enhance-the-Poster | with-cue | A + C poster with-cue |

四種 counterbalance 組合都可用同一套影片處理：

| Group pattern | Task 1 播放 | Task 2 播放 |
| --- | --- | --- |
| Lost-at-Sea no-cue -> Enhance-the-Poster with-cue | A + `B_lost_at_sea_no_cue` | `C_enhance_the_poster_with_cue` |
| Lost-at-Sea with-cue -> Enhance-the-Poster no-cue | A + `B_lost_at_sea_with_cue` | `C_enhance_the_poster_no_cue` |
| Enhance-the-Poster no-cue -> Lost-at-Sea with-cue | A + `C_enhance_the_poster_no_cue` | `B_lost_at_sea_with_cue` |
| Enhance-the-Poster with-cue -> Lost-at-Sea no-cue | A + `C_enhance_the_poster_with_cue` | `B_lost_at_sea_no_cue` |

這裡的「獨立」指的是 B/C 每支任務影片都已經完整包含該任務與該 condition 的說明，不需要再剪接 no-cue 或 with-cue 片段；但共通系統操作仍由 A 影片負責。因此第一輪播放 A + 任務影片，第二輪只播放下一支任務影片。

## 1. 影片 A：共通系統介紹

### A.1 開場

各位好，接下來會介紹今天使用的線上會議輔助系統。

今天你們會使用這個系統完成小組任務。每一輪任務都會先有個人思考階段，再有小組公開討論階段，最後會有一小段時間讓你調整自己的個人排序。

今天不是能力測驗，也沒有個人分數。我們關心的是小組討論時，大家如何先形成自己的想法，如何公開討論，以及系統介面是否能幫助你整理和回顧自己的想法。

過程中可能會使用指定的會議或錄影工具，並依研究設定保存與任務及系統互動相關的資料，例如作答內容、討論紀錄、想法筆記、文字訊息、系統提示與操作紀錄。請先關閉和任務無關的私人視窗與通知。

### A.2 主畫面

進入系統後，你會看到幾個主要功能區域；實際位置、名稱與呈現方式可能依系統版本調整。

其中一個區域是任務工作區。這裡會顯示任務說明、任務材料，以及目前需要操作的內容。

畫面中也會有會議或討論區，供大家進行公開討論；實際使用的會議工具以當時畫面為準。

另外會有個人輔助區，可能包含討論紀錄、idea blocks 和文字交流等功能。這些功能可協助你回顧內容、整理自己的想法，或以文字參與討論；實際分頁與名稱以當時介面為準。

畫面上也會顯示目前階段和倒數時間。請以主持人的說明和畫面上的階段為準。

### A.3 公開發言與悄悄話

介面中會提供公開發言和記錄個人想法的控制方式；按鈕名稱與位置可能依系統版本調整。

使用公開發言功能時，你的聲音會進入小組討論，其他人可以聽到，相關內容也可能依研究設定被記錄。

使用個人想法記錄功能時，你的聲音不會透過公開會議傳給其他人，系統可能將內容保留為個人筆記，或進一步整理成 idea blocks。這個功能適合在思考時記下尚未確定的理由、疑問或其他內容。

你可以用 Space 快捷鍵切換公開發言和悄悄話。使用時請注意目前按鈕狀態，確認自己是在公開講話，還是在記錄私人想法。

### A.4 idea blocks 與文字交流

idea blocks 是系統可能根據你的語音、個人想法紀錄或文字輸入整理出的內容。你可以把它當作自己的思考筆記；實際名稱與呈現方式以當時介面為準。

idea blocks 可以包含理由、疑問、不同意見、支持內容、關鍵字，或還沒有整理好的想法。

如果介面提供手動新增功能，你也可以自行輸入想法，並依畫面指示加入個人筆記清單。

依系統當時提供的功能，你可以查看、整理或調整這些想法。

請注意，個人想法原則上不會自動公開給其他參與者。只有當你主動透過系統提供的分享方式送出，或自己公開講出來，其他人才會看到或聽到。

文字交流區可作為公開發言管道之一。如果你不想用口頭發言，也可以輸入文字讓其他人看到；實際送出、換行和其他操作方式請依畫面提示。

### A.5 共通收尾

每一輪任務開始前，任務影片會再說明該任務的內容、預計時間，以及這一輪是否可能顯示系統提示。

若遇到頁面、裝置權限或操作上的技術問題，可以告訴現場協助人員；但任務內容和判斷請依照你自己的想法完成。

## 2. 影片 B1：Lost-at-Sea no-cue

### B1.1 開場與 condition

接下來這一輪任務是 Lost-at-Sea。

這一輪不會顯示想法關聯提示，也就是系統不會主動提醒你可能有人和你有相近想法。

你仍然可以使用這一輪提供的基本功能，例如個人作答、小組共同作答、公開發言、個人想法紀錄、討論回顧、想法筆記和文字交流。

請記得，個人想法筆記不會自動公開。你可以把它當作自己的筆記；如果想把某個想法帶進小組討論，可以自己公開說出來，或透過系統提供的公開分享方式送出。

### B1.2 任務情境

你們搭乘一艘私人遊艇，在南太平洋上漂流。因為一場原因不明的火災，遊艇和船上大部分物品都被燒毀，遊艇正在慢慢下沉。

由於重要的導航設備被破壞，而且大家在滅火時分心，你們無法確定目前位置。你們目前最好的估計是：距離最近的陸地約 1,600 公里。

在遊艇完全沉沒前，你們成功救出一艘可用的橡皮救生艇和船槳，救生艇足以載下所有人和 15 項物品。這些物品都沒有受損，可以正常使用，但不能連上網路，也不能直接呼叫救援。

你們的任務是依照這些物品對海上求生的重要性排序。第 1 名是最重要，第 15 名是最不重要。

這個任務請不要上網查答案。重點不是測驗誰知道標準答案，而是觀察小組如何提出理由、討論、說服、妥協，形成共同決定。

### B1.3 操作與時間

這一輪會有三個階段。

第一個是個人思考階段，預計時間是 8 分鐘。請先自己閱讀情境，根據你認為對海上求生的重要性，把 15 項物品排出自己的順序。這個階段請先不要和其他人討論。

在 Private Phase 中，請盡量邊想邊說出理由，或用文字記錄理由，讓系統幫你形成 Idea Blocks。你也可以點物品圖片查看放大圖和簡短說明。

第二個是小組討論階段，預計時間是 20 分鐘。這個階段請和小組成員公開討論，完成一份小組共同排序。小組共同作答不一定要完全符合任何一個人的個人排序，但每個人的意見都應該被聽見。

即使這一輪沒有系統提示，也請你依照自己的判斷，在合適的時候提出理由、疑問、支持或不同意見。

第三個是個人回顧階段，預計時間是 2 分鐘。這個階段請不要再公開討論，只要根據剛才討論調整或維持自己的個人排序。

### B1.4 收尾

這一輪任務請把重點放在理由和討論過程。你可以改變想法，也可以維持自己的排序。沒有標準發言方式，只要依照你自己的判斷參與討論即可。

## 4. 影片 C1：Enhance-the-Poster no-cue

### C1.1 開場與 condition

接下來這一輪任務是 Enhance-the-Poster。

這一輪不會顯示想法關聯提示，也就是系統不會主動提醒你可能有人和你有相近想法。

你仍然可以使用這一輪提供的基本功能，例如建立改善項目、個人作答、小組共同作答、公開發言、個人想法紀錄、討論回顧、想法筆記和文字交流。

請記得，個人想法筆記不會自動公開。你可以把它當作自己的筆記；如果想把某個改善想法帶進小組討論，可以自己公開說出來，或透過系統提供的公開分享方式送出。

### C1.2 任務目標

你們會看到一份 Canva 材料。裡面包含淨灘活動招募海報、任務說明與任務需求、reviewer feedback 摘要、圖床、可替換的文字與 slogan library，以及海報元件和改善動作的 library。

海報本身包含淨灘插圖、主標語、日期時間、地點、活動說明，以及報名 QR Code。Reviewer feedback 不需要逐字朗讀；請把它當作判斷哪些地方需要優先改善的參考。

這個任務不是要你直接修改原始海報，也不是測試美編能力。你的任務是在線上會議輔助系統中建立改善項目，例如指出「希望調整哪個部分，以及如何調整」。

你可以從內容是否容易理解、資訊是否清楚、行動方式是否明顯、視覺呈現是否有效，以及整體資訊是否足以支持使用者做決定等方向思考。

### C1.3 操作與時間

這一輪會有四個階段。

第一個是 Private Phase 1，時間是 5 分鐘。請先自己看 Canva 材料，綜合海報內容、reviewer feedback 摘要、圖床、文字與 slogan library，以及系統提供的海報元件和改善動作，組合改善項目。每個人至少建立 4 個，想建立更多也可以。

建立改善項目時，請依介面提供的方式選擇或描述想調整的部分與改善方向，系統會將內容整理成一個項目。

例如，你可以提出放大行動入口、調整主標題呈現方式，或其他有助於改善內容的做法。如果系統要求補充資訊，請依照畫面輸入。

建立後，可依系統當時提供的功能整理或調整項目。請注意，這個階段仍然是個人思考時間，請先不要和其他人討論。

第二個是 Private Phase 2，時間是 5 分鐘。系統會整理大家建立的改善項目，請你先做自己的個人排序。你只需要把最重要的改善項目排到前面。

第三個是 Public Phase，時間是 20 分鐘。這個階段請和小組成員公開討論，選出最多 15 個最重要的改善項目，並排出小組共同優先順序。第 16 個之後代表暫時不會改動。

小組討論階段中，可以使用系統提供的語音或文字方式與其他人討論。請依主持人或當次任務的參與要求進行公開發言。

即使這一輪沒有系統提示，也請你依照自己的判斷，在合適的時候提出理由、疑問、支持或不同意見。

第四個是個人回顧階段，預計時間是 2 分鐘。這個階段請不要再公開討論，只要根據剛才討論調整或維持自己的個人判斷。

### C1.4 收尾

這一輪任務請把重點放在具體改善項目和排序理由。你不需要真的修改海報，也不需要追求唯一正確答案。請依照你自己的觀察，提出你認為最值得優先改善的地方。

第二個是 Private Phase 2，時間是 5 分鐘。系統會整理大家建立的改善項目，請你先做自己的個人排序。你只需要把最重要的改善項目排到前面。

第三個是 Public Phase，時間是 20 分鐘。這個階段請和小組成員公開討論，選出最多 15 個最重要的改善項目，並排出小組共同優先順序。第 16 個之後代表暫時不會改動。

小組討論階段中，可以使用系統提供的語音或文字方式與其他人討論。請依主持人或當次任務的參與要求進行公開發言。

小組討論階段中，系統可能提示他人的改善方向或理由與你相近或有所不同。這些提示僅供參考，你可以自行決定是否把相關想法帶進討論。

第四個是個人回顧階段，預計時間是 2 分鐘。這個階段請不要再公開討論，只要根據剛才討論調整或維持自己的個人判斷。

### C2.4 收尾

這一輪任務請把重點放在具體改善項目和判斷理由。系統提示只是輔助提醒，不是標準答案。你不需要真的修改海報，也不需要追求唯一正確答案；請依照你自己的觀察參與討論。
