# OmniObserve 教學影片講稿

本文件是 OmniObserve 正式實驗影片講稿的唯一 source of truth。內容包含任務前教學影片與最後 Debrief 影片：

1. `A_common_system_intro`：共通系統介紹。
2. `B_lost_at_sea_no_cue`：Lost-at-Sea，no-cue 版本。
3. `B_lost_at_sea_with_cue`：Lost-at-Sea，with-cue 版本。
4. `C_enhance_the_poster_no_cue`：Enhance-the-Poster，no-cue 版本。
5. `C_enhance_the_poster_with_cue`：Enhance-the-Poster，with-cue 版本。
6. `D_final_debrief`：最後 Debrief，只能在三位真實參與者個別訪談都完成後播放。

錄影時不要說「實驗組 / 對照組」。面向參與者時，用「這一輪不會顯示系統提示」和「這一輪可能會顯示系統提示」即可。

## 0. 現場播放規則與 counterbalance 矩陣

- 每場一開始播放共通系統介紹 `A_common_system_intro`。
- 每輪任務開始前，只播放該任務、該 condition 對應的一支任務影片。
- 不要在同一輪任務前同時播放 no-cue 和 with-cue 影片。
- 任務影片一律說「這一輪任務」，不要說「第一個任務」或「第二個任務」。
- 若某任務是第二輪，不需要重播共通系統介紹，只播放該任務影片即可。
- 任務前影片中不要提暗樁、研究假設、後台判斷規則或 Lost-at-Sea 專家答案。
- 最後 Debrief 影片只能在三位真實參與者的個別訪談都完成後播放。

| 任務 | Condition | 播放影片 |
| --- | --- | --- |
| Lost-at-Sea | no-cue | A + B lost no-cue |
| Lost-at-Sea | with-cue | A + B lost with-cue |
| Enhance-the-Poster | no-cue | A + C poster no-cue |
| Enhance-the-Poster | with-cue | A + C poster with-cue |

四種 counterbalance 組合都可用同一套影片處理：

| Group pattern | Task 1 播放 | Task 2 播放 |
| --- | --- | --- |
| Lost-at-Sea no-cue -> Enhance-the-Poster with-cue | A + `B_lost_at_sea_no_cue` | `C_enhance_the_poster_with_cue` |
| Lost-at-Sea with-cue -> Enhance-the-Poster no-cue | A + `B_lost_at_sea_with_cue` | `C_enhance_the_poster_no_cue` |
| Enhance-the-Poster no-cue -> Lost-at-Sea with-cue | A + `C_enhance_the_poster_no_cue` | `B_lost_at_sea_with_cue` |
| Enhance-the-Poster with-cue -> Lost-at-Sea no-cue | A + `C_enhance_the_poster_with_cue` | `B_lost_at_sea_no_cue` |

這裡的「獨立」指的是 B/C 每支任務影片都已經完整包含該任務與該 condition 的說明，不需要再剪接 no-cue 或 with-cue 片段；但共通系統操作仍由 A 影片負責。因此第一輪播放 A + 任務影片，第二輪只播放下一支任務影片。

## 1. 影片 A：共通系統介紹

### A.1 開場

各位好，接下來會介紹今天使用的 OmniObserve 系統。

今天你們會使用這個系統完成小組任務。每一輪任務都會先有個人思考階段，再有小組公開討論階段，最後會有一小段時間讓你調整自己的個人排序。

今天不是能力測驗，也沒有個人分數。我們關心的是小組討論時，大家如何先形成自己的想法，如何公開討論，以及系統介面是否能幫助你整理和回顧自己的想法。

過程中會使用 Google Meet 錄影，也會保存系統中的排序、逐字稿、Idea Blocks、聊天室訊息、提示紀錄和操作紀錄。請先關閉和任務無關的私人視窗與通知。

### A.2 主畫面

進入系統後，你會看到幾個主要區域。

左側最大的區域是任務工作區。這裡會顯示任務說明、任務材料，以及你目前需要操作的排序或項目清單。

畫面下方或中間會有 Jitsi 視訊會議區，這是大家公開討論時使用的會議空間。

右側是 Private Board。它有三個分頁：逐字稿、Idea Blocks、聊天室。逐字稿用來看系統記錄到的發言內容；Idea Blocks 用來整理你自己的想法；聊天室是公開文字發言區。

畫面上也會顯示目前階段和倒數時間。請以主持人的說明和畫面上的階段為準。

### A.3 公開發言與悄悄話

畫面下方有兩個重要按鈕：「公開發言」和「悄悄話」。

如果你按「公開發言」，你的聲音會進入小組討論，其他人可以聽到，這些內容也會被記錄為公開討論內容。

如果你按「悄悄話」，你的聲音不會透過 Jitsi 公開給其他人聽到，但系統會把它當作你自己的私人想法來記錄，之後可能整理成 Idea Blocks。這個功能適合在自己思考時邊想邊說，也適合先把還不確定的理由或疑問記下來。

你可以用 Space 快捷鍵切換公開發言和悄悄話。使用時請注意目前按鈕狀態，確認自己是在公開講話，還是在記錄私人想法。

今天 private phase 會請你 think aloud，也就是在自己做判斷時，把正在考慮的理由說出來。這不是要你講完整答案，也不是要你說服其他人；你可以說「我正在比較這兩個」「我暫時這樣排，因為...」這類還在思考中的理由。如果出聲不自然，也可以用系統裡的文字輸入把理由記下來。

### A.4 Idea Blocks 與聊天室

Idea Blocks 是系統根據你的語音、悄悄話或文字輸入整理出的想法卡片。你可以把它當作自己的思考筆記。

Idea Blocks 可以是理由、疑問、不同意見、支持理由、關鍵字，或還沒有整理好的想法。

你也可以在 Idea Blocks 分頁下方手動輸入 idea block，按新增後就會出現在清單中。

每個 idea block 可以展開查看，也可以編輯或刪除。

請注意，Idea Blocks 不會自動公開給其他參與者。只有當你在 Public Phase 主動把 idea block 送到聊天室，或你自己公開講出來，其他人才會看到或聽到。

聊天室是正式公開發言管道。如果你不想用口頭發言，也可以在聊天室輸入文字，讓所有人看到。Enter 可以送出，Shift 加 Enter 可以換行。

### A.5 共通收尾

每一輪任務開始前，任務影片會再說明該任務的內容、時間，以及這一輪是否會顯示系統提示。

若遇到頁面壞掉、麥克風權限、按鈕找不到等技術問題，可以告訴研究人員；但任務內容和排序判斷請依照你自己的想法完成。

## 2. 影片 B1：Lost-at-Sea no-cue

### B1.1 開場與 condition

接下來這一輪任務是 Lost-at-Sea。

這一輪不會顯示 Similarity Cue，也就是系統不會跳出「有人和你有相近想法」這類提示。

你仍然可以使用所有基本功能：個人排序、Public Ranking、公開發言、悄悄話、逐字稿、Idea Blocks 和聊天室。

請記得，Idea Blocks 不會自動公開。你可以把它當作自己的筆記；如果你想把某個想法帶進小組討論，可以自己公開說出來，或在 Public Phase 把它送到聊天室。

### B1.2 任務情境

你們搭乘一艘私人遊艇，在南太平洋上漂流。因為一場原因不明的火災，遊艇和船上大部分物品都被燒毀，遊艇正在慢慢下沉。

由於重要的導航設備被破壞，而且大家在滅火時分心，你們無法確定目前位置。你們目前最好的估計是：距離最近的陸地約 1,600 公里。

在遊艇完全沉沒前，你們成功救出一艘可用的橡皮救生艇和船槳，救生艇足以載下所有人和 15 項物品。這些物品都沒有受損，可以正常使用，但不能連上網路，也不能直接呼叫救援。

你們的任務是依照這些物品對海上求生的重要性排序。第 1 名是最重要，第 15 名是最不重要。

這個任務請不要上網查答案。重點不是測驗誰知道標準答案，而是觀察小組如何提出理由、討論、說服、妥協，形成共同決定。

### B1.3 操作與時間

這一輪會有三個階段。

第一個是 Private Phase，時間是 8 分鐘。請先自己閱讀情境，根據你認為對海上求生的重要性，把 15 項物品排出自己的順序。這個階段請先不要和其他人討論。

在 Private Phase 中，請盡量邊想邊說出正在比較和排序的理由，讓系統幫你形成 Idea Blocks。你不需要講得完整，也不需要公開給其他人聽；如果不想出聲，也可以用文字記錄理由。你也可以點物品圖片查看放大圖和簡短說明。

第二個是 Public Phase，時間是 20 分鐘。這個階段請和小組成員公開討論，完成一份小組共同排序。Public Ranking 是小組共同答案，不一定要完全符合任何一個人的個人排序，但每個人的意見都應該被聽見。

即使這一輪沒有系統提示，也請你依照自己的判斷，在合適的時候提出理由、疑問、支持或不同意見。

第三個是 Reflect Phase，時間是 2 分鐘。這個階段請不要再公開討論，只要根據剛才討論調整或維持自己的個人排序。

### B1.4 收尾

這一輪任務請把重點放在理由和討論過程。你可以改變想法，也可以維持自己的排序。沒有標準發言方式，只要依照你自己的判斷參與討論即可。

## 4. 影片 C1：Enhance-the-Poster no-cue

### C1.1 開場與 condition

接下來這一輪任務是 Enhance-the-Poster。

這一輪不會顯示 Similarity Cue，也就是系統不會跳出「有人和你有相近想法」這類提示。

你仍然可以使用所有基本功能：建立改善項目、個人排序、Public Ranking、公開發言、悄悄話、逐字稿、Idea Blocks 和聊天室。

請記得，Idea Blocks 不會自動公開。你可以把它當作自己的筆記；如果你想把某個改善想法帶進小組討論，可以自己公開說出來，或在 Public Phase 把它送到聊天室。

### C1.2 任務目標

你們會看到一份 Canva 材料。裡面包含淨灘活動招募海報、任務說明與任務需求、圖床、可替換的文字與 slogan library，以及海報元件和改善動作的 library。

海報本身包含淨灘插圖、主標語、日期時間、地點、活動說明，以及報名 QR Code。若這一輪 Canva 材料有 reviewer feedback 摘要，請把它當作判斷哪些地方需要優先改善的參考；若沒有，請直接依海報內容、任務需求、素材與自己的判斷建立改善項目。

這個任務不是要你直接修改 Canva 海報，也不是測試美編能力。你的任務是在 OmniObserve 裡用文字建立改善項目，也就是「對哪個海報元件做什麼修改」。

你可以考慮：路過的人能不能快速理解活動內容、會不會願意報名、時間地點是否清楚、QR Code 是否容易被注意和掃描、視覺層次是否有效，以及資訊是否足以讓人安心參與。

### C1.3 操作與時間

這一輪會有四個階段。

第一個是 Private Phase 1，時間是 5 分鐘。請先自己看 Canva 材料，綜合海報內容、任務需求、圖床、文字與 slogan library，以及系統提供的海報元件和改善動作，組合改善項目；若材料有 reviewer feedback，也可以納入考量。每個人至少建立 4 個，想建立更多也可以。建立時請把你正在考慮的理由說出來；如果不想出聲，也可以用文字輸入記錄。

建立改善項目時，請先選一個海報元件，再選一個改善動作。系統會把它組合成一個 task item。

例如，你可以選擇「QR 碼」加上「放大」，形成一個改善項目；也可以選擇「主標題」加上「改顏色」。如果系統提供需要填編號或補充說明的動作，請依照畫面輸入。

建立後，你可以編輯、刪除，或調整項目的優先順序。請注意，這個階段仍然是個人思考時間，請先不要和其他人討論。

第二個是 Private Phase 2，時間是 7 分鐘。系統會整理大家建立的改善項目，請你先做自己的個人排序。這個階段一定要完成你的個人排序；如果時間快到，請先把你認為最重要的改善項目排到前面。

第三個是 Public Phase，時間是 20 分鐘。這個階段請和小組成員公開討論，選出最多 15 個最重要的改善項目，並排出小組共同優先順序。第 16 個之後代表暫時不會改動。

Public Phase 中可以開啟公開麥克風和其他人討論；每位參與者請至少公開發言兩次。口頭發言或在聊天室公開送出訊息，都算公開發言。

即使這一輪沒有系統提示，也請你依照自己的判斷，在合適的時候提出理由、疑問、支持或不同意見。

第四個是 Reflect Phase，時間是 2 分鐘。這個階段請不要再公開討論，只要根據剛才討論調整或維持自己的個人排序。

### C1.4 收尾

這一輪任務請把重點放在具體改善項目和排序理由。你不需要真的修改海報，也不需要追求唯一正確答案。請依照你自己的觀察，提出你認為最值得優先改善的地方。

### C2.3 操作與時間

這一輪會有四個階段。

第一個是 Private Phase 1，時間是 5 分鐘。請先自己看 Canva 材料，綜合海報內容、任務需求、圖床、文字與 slogan library，以及系統提供的海報元件和改善動作，組合改善項目；若材料有 reviewer feedback，也可以納入考量。每個人至少建立 4 個，想建立更多也可以。建立時請把你正在考慮的理由說出來；如果不想出聲，也可以用文字輸入記錄。

建立改善項目時，請先選一個海報元件，再選一個改善動作。系統會把它組合成一個 task item。

例如，你可以選擇「QR 碼」加上「放大」，形成一個改善項目；也可以選擇「主標題」加上「改顏色」。如果系統提供需要填編號或補充說明的動作，請依照畫面輸入。

建立後，你可以編輯、刪除，或調整項目的優先順序。請注意，這個階段仍然是個人思考時間，請先不要和其他人討論。

第二個是 Private Phase 2，時間是 7 分鐘。系統會整理大家建立的改善項目，請你先做自己的個人排序。這個階段一定要完成你的個人排序；如果時間快到，請先把你認為最重要的改善項目排到前面。

第三個是 Public Phase，時間是 20 分鐘。這個階段請和小組成員公開討論，選出最多 15 個最重要的改善項目，並排出小組共同優先順序。第 16 個之後代表暫時不會改動。

Public Phase 中可以開啟公開麥克風和其他人討論；每位參與者請至少公開發言兩次。口頭發言或在聊天室公開送出訊息，都算公開發言。

Public Phase 中，如果看到 same reason，代表可能有人和你想改的方向相近，而且理由也相近。如果看到 different reason，代表可能有人同樣認為某個海報元件值得改，但背後理由不同。你可以自行決定是否把這個想法帶進討論。

第四個是 Reflect Phase，時間是 2 分鐘。這個階段請不要再公開討論，只要根據剛才討論調整或維持自己的個人排序。

### C2.4 收尾

這一輪任務請把重點放在具體改善項目和排序理由。Cue 只是輔助提醒，不是標準答案。你不需要真的修改海報，也不需要追求唯一正確答案；請依照你自己的觀察參與討論。

## 6. 影片 D：最後 Debrief

本影片只在三位真實參與者的個別訪談都完成後播放。不得在任務前、兩個 task 中間、task 後小表單前後、或任何個別訪談前播放。

建議檔名：

- `D_final_debrief.mp4`

影片講稿：

> 各位好，謝謝你們完成今天的兩個任務與個別訪談。
>
> 最後我們要補充說明今天實驗設計的一部分：四位參與者中有一位是研究團隊安排的成員。這位成員的角色是協助我們讓討論中出現比較一致或有壓力的討論情境，讓我們觀察系統是否能幫助其他參與者回顧自己的想法、判斷是否要公開提出。
>
> 這不是要評價任何人的能力，也不是要判斷誰的答案對錯。我們關心的是整體討論過程，以及系統在 private phase、public phase、cue、Idea Blocks 和排序過程中是否能支援參與者整理與表達自己的想法。
>
> Debrief 之後不會再進行研究訪談。如果你對剛才說明、資料使用或實驗流程有任何疑問，可以現在詢問，也可以之後直接聯絡研究團隊。
