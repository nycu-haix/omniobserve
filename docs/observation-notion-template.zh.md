# OmniObserve 觀察筆記 Notion 模板設計

三位研究者各自在不同教室，同時在同一個 Notion 頁面上記錄。本文件說明模板的頁面結構與各欄位定義。

---

## 頁面結構總覽

```
📄 [Session ID] 觀察筆記
├── 🔷 Session 基本資訊          ← 實驗開始前填一次
├── 📋 快速注意事項              ← 觀察者參考用，唯讀
├── 🗃️ Critical Incident 記錄    ← 三位觀察者同時填入的 Database
└── 💬 事後訪談記錄
    ├── P1 訪談
    ├── P2 訪談
    └── P3 訪談
```

---

## 🔷 Session 基本資訊

頁面頂部的普通文字區塊，實驗開始前由主持人填寫一次。

| 欄位 | 說明 | 範例 |
| --- | --- | --- |
| Session ID | 本次實驗編號 | `S01` |
| 日期 | | `2025-07-10` |
| 實驗條件 | 對照組 / 實驗組 | `實驗組` |
| P1 房間研究者 | | `小明` |
| P2 房間研究者 | | `小華` |
| P3 房間研究者 | | `小美` |
| 備註 | 特殊情況、設備問題等 | |

---

## 📋 快速注意事項

頁面中段的 Callout 區塊，唯讀，提醒觀察者要特別注意的行為。

> **記錄這些事件：**
> - 參與者看到 cue 後不久發言
> - 參與者看到 cue 但忽略
> - 參與者有 private idea，但整場公開討論都沒有提出
> - 參與者看起來想發言但停住
> - 參與者因其他人強勢論點而修改排序
> - 參與者不理解 mic state、cue 意義或排序操作
> - 參與者求助或看起來被介面卡住
> - 參與者用 group chat 取代公開語音發言

## 🗃️ Critical Incident 記錄 Database

### Database 屬性定義

| 屬性名稱 | 類型 | 選項 / 說明 |
| --- | --- | --- |
| **事件摘要** | Title | 一句話描述，例如「P2 看到 cue 後停頓未發言」 |
| **時間戳記** | Text | 實驗計時器時間，格式 `mm:ss`，例如 `06:30` |
| **觀察者** | Select | `R1（P1 房間）` / `R2（P2 房間）` / `R3（P3 房間）` |
| **參與者** | Select | `P1` / `P2` / `P3` |
| **事件類型** | Multi-select | 見下方說明 |
| **事件描述** | Text | 具體行為的客觀描述 |
| **可能解釋** | Text | 觀察者推測的原因 |
| **後續訪談問題** | Text | 建議在事後訪談中追問的問題 |
| **優先追問** | Checkbox | 勾選代表這個 incident 一定要在訪談中問到 |

### 事件類型選項（Multi-select）

| 標籤 | 說明 |
| --- | --- |
| `Cue 觸發發言` | 看到 cue 後有發言行為 |
| `Cue 被忽略` | 看到 cue 但沒有後續行為 |
| `Private-to-public 失敗` | private 階段有提到，公開討論沒有出現 |
| `猶豫未發言` | 明顯想說話但停住 |
| `社會影響改排序` | 因他人論點修改自己排序 |
| `介面操作困難` | mic state、cue、排序等操作不熟悉或卡住 |
| `求助` | 主動向研究者求助 |
| `改用文字` | 用 group chat 取代語音發言 |
| `其他` | 不在上述分類內的值得記錄事件 |

### Database 預設視圖

建議建立兩個視圖：

1. **Table 視圖（預設）**：所有觀察者的 incident 按時間戳記排序，便於事後統整。
2. **依觀察者分組視圖**：以「觀察者」欄位分組，方便各房間研究者只看自己的記錄。

---

## 操作說明

1. 實驗開始前，主持人在 Session 基本資訊填入當天資訊。
2. 三位研究者各自在自己的教室，用手機或備用螢幕開啟同一個 Notion 頁面。
3. 觀察到 incident 時，直接在 Database 新增一筆記錄，填入時間戳記、參與者、事件描述即可；可能解釋與訪談問題可在實驗結束後補充。
4. 實驗結束後統一檢閱所有記錄，勾選「優先追問」，作為訪談的提問依據。
5. 事後訪談由負責該參與者所在教室的研究者進行，訪談結果直接在對應的 Px 訪談頁面上記錄。

## 💬 事後訪談記錄

三位參與者分別由其所在教室的研究者訪談，每人約 8 到 12 分鐘。訪談選用各自的小頁面記錄，以下為標準模板，三份內容相同。

> 提醒：不要一開始就說本研究在測試 cue 是否讓參與者更願意發言。先問整體經驗，再追問具體事件。
