# 資料命名與輸出規格

本文件定義正式實驗資料命名、輸出項目與最低檢查標準。

## 1. 命名原則

建議 session folder：

```text
data/{date}_{group_id}/
```

範例：

```text
data/2026-06-18_G01/
```

檔名格式：

```text
{group_id}_{task}_{condition}_{artifact}_{participant_or_scope}.{ext}
```

欄位：

- `group_id`: `G01`, `G02`
- `task`: `lost_at_sea`, `enhance_poster`
- `condition`: `no_cue`, `with_cue`
- `artifact`: 見下表
- `participant_or_scope`: 真實參與者用 `G01P1`, `G01P2`, `G01P3`；也可用 `group`, `all`, `observer1`
- 暗樁使用內部 Confederate ID 記錄，不出現在受試者可見畫面。
- 目前系統 URL participant id 仍要求整數；每場需保存一份私下 mapping table，記錄 `system_id 1/2/3/4 -> research participant code / Confederate internal ID`。

## 2. 必要輸出

| Artifact | Scope | Example |
| --- | --- | --- |
| participant mapping table | group/internal | `G01_all_all_participant_mapping_internal.csv` |
| initial personal ranking | participant | `G01_lost_at_sea_no_cue_initial_ranking_G01P1.csv` |
| final personal ranking | participant | `G01_lost_at_sea_no_cue_final_ranking_G01P1.csv` |
| group ranking | group | `G01_lost_at_sea_no_cue_group_ranking_group.csv` |
| transcript | participant/group/interview | `G01_enhance_poster_with_cue_public_transcript_group.txt` |
| idea blocks | participant/all | `G01_enhance_poster_with_cue_idea_blocks_all.csv` |
| cue logs | group | `G01_enhance_poster_with_cue_cue_logs_group.csv` |
| public chat logs | group | `G01_lost_at_sea_no_cue_public_chat_group.csv` |
| screen recording workaround | participant | `G01_all_all_google_meet_recording_G01P1.mp4` |
| participant familiarity screener | group/internal | `G01_all_all_familiarity_screener_internal.csv` |
| small form response | participant | `G01_lost_at_sea_no_cue_small_form_G01P1.csv` |
| final form response | participant | `G01_all_all_final_form_G01P1.csv` |
| observer notes | observer | `G01_all_all_observer_notes_observer1.csv` |
| interview recording | participant | `G01_all_all_interview_G01P1.m4a` |
| interview transcript | participant | `G01_all_all_interview_transcript_G01P1.txt` |
| session summary | group | `G01_all_all_session_summary_group.md` |
| incident log | group | `G01_all_all_incident_log_group.csv` |

For no-cue task, cue log file may exist with `cue_enabled=false` and zero cue events, or be documented in session summary as not enabled.

Idea block export must include a source marker when available:

- `source_type`: `speech` / `text` / `mixed` / `unknown`
- `source_ref`: transcript segment ID, public chat message ID, action item ID, ranking note ID, or another system-local reference
- `participant_id`
- `task`
- `condition`
- `phase`

Google Forms responses and interview notes that ask about condition-specific features must include skip metadata:

- `idea_blocks_available`
- `idea_blocks_seen`
- `idea_blocks_used`
- `review_feedback_available`
- `review_feedback_used`
- `similarity_cue_available`
- `similarity_cue_seen`
- `not_applicable_reason`

Valid `not_applicable_reason` values include `not_available`, `did_not_use`, `did_not_see`, `no_cue_seen`, `no_cue_by_condition`, `technical_issue`, and `unsure`.

## 3. Session Summary 最低內容

每場實驗建立一份 `session_summary`，至少包含：

- group ID。
- 日期、地點。
- Host/Admin。
- Observer 對應 participant。
- Participant mapping table: system integer id 對應 `G01P1` / `G01P2` / `G01P3` / Confederate internal ID。
- Confederate ID，僅內部記錄。
- task order。
- condition assignment。
- system version / commit。
- Canva material version。
- Google Forms versions / edit URLs。
- Feature availability metadata for each task: idea blocks, reviewer feedback, and similarity cue availability/use status.
- group familiarity level、recruitment decision、over-familiar caveat。
- 每位真實參與者個別 Google Meet recording file link or storage path；這是外部錄影 workaround，不是 OmniObserve 系統輸出。
- phase 實際開始/結束時間。
- speech/text idea block capture 是否正常。
- 是否發生系統故障、錄影問題、表單漏填、cue 異常、參與者中途退出。
- 檔案清單與缺漏檢查。

## 4. Incident Log 欄位

| 欄位 | 說明 |
| --- | --- |
| group_id | 場次 |
| timestamp | HH:MM:SS |
| task | lost_at_sea / enhance_poster / all |
| condition | no_cue / with_cue / all |
| phase | setup / private / public / reflect / form / interview / debrief |
| actor | host / admin / observer / participant_id / system |
| incident_type | system_error / recording_issue / cue_issue / participant_confusion / protocol_deviation / form_issue / other |
| description | 具體描述 |
| action_taken | 當下處理 |
| data_impact | 對資料的影響 |
| followup_needed | 是否需後續處理 |

## 5. Observer Note 表單欄位

Observer notes 建議用表單或試算表收集。

| 欄位 | 說明 |
| --- | --- |
| group_id | 場次 |
| observer_id | 觀察者 |
| participant_id | 負責真實參與者 |
| task | lost_at_sea / enhance_poster |
| condition | no_cue / with_cue |
| phase | private / public / reflect / form / interview |
| timestamp | HH:MM:SS |
| event_type | private_idea / public_speech / public_chat / cue_reaction / ranking_change / facilitator_prompt / confusion / incident |
| related_item | Lost item 或 Poster component/action |
| cue_type | none / same_reason / different_reason |
| reasoning_capture_mode | speech / text / both / none |
| observed_event | 具體看到/聽到的事件 |
| interpretation | 觀察者推測，若沒有則留空 |
| followup_question | 訪談要追問 |
| group_familiarity_context | none / low / medium / high / mixed / unknown |
| familiarity_followup_needed | yes / no |

## 6. Familiarity / Analysis Covariate 欄位

若使用表單或試算表彙整，至少保留：

| 欄位 | 說明 |
| --- | --- |
| group_familiarity_level | none / low / medium / high / mixed / unknown |
| over_familiar_group | true / false / unknown |
| known_members_count | 0 / 1 / 2 / 3 / unknown |
| collaboration_frequency_last_3_months | never / once_or_twice / monthly / weekly / multiple_times_per_week / unknown |
| public_disagreement_comfort | 1-7；若每 task 填一次，需保留 task 與 condition |
| familiarity_effect_on_private_channel | reduced_need / no_obvious_effect / increased_need / mixed / unknown |
| analysis_caveat | 對 private-channel usage、cue usage 或 public disagreement 的分析限制 |

## 7. 最低完成檢查

每場完成後檢查：

- [ ] 三位真實參與者都有兩個 task 的 initial/final personal ranking。
- [ ] 兩個 task 都有 group ranking。
- [ ] Poster group ranking 不超過 15 items。
- [ ] Lost group ranking 包含 15 items。
- [ ] 兩個 task 都有 public discussion transcript；private reasoning capture 可由 speech transcript and/or typed reasoning 滿足。
- [ ] 兩個 task 都有 idea blocks。
- [ ] idea blocks 可區分 speech / text 來源，或在 session summary 記錄無法區分的原因。
- [ ] with-cue task 有 cue logs。
- [ ] no-cue task 明確標記 cue disabled。
- [ ] 兩個 task 都有 public chat logs。
- [ ] 三位真實參與者各自的個別 Google Meet recording 已保存。
- [ ] participant mapping table 已保存於內部資料夾。
- [ ] familiarity screener 已保存，session summary 已標記 group familiarity level。
- [ ] 三位真實參與者各有兩份小表單 response。
- [ ] 三位真實參與者各有大表單 response。
- [ ] 表單 response 中所有 idea blocks、reviewer feedback、cue 題目都有 availability/use 或 `N/A` 原因。
- [ ] 三份個別訪談錄音/逐字稿保存。
- [ ] Observer notes 收齊。
- [ ] session summary 完成。
- [ ] incident log 完成，即使無 incident 也寫 `none`。
