# 問卷與訪談題綱

本文件是正式實驗的問卷與個別訪談題綱。三份 Google Forms 仍是實際填答來源；本文件用來確保研究人員知道每個欄位的目的、何時詢問，以及哪些追問不得洩露暗樁或研究假設。

## 1. 受試者熟悉度紀錄

Private-expression、whisper、idea blocks 與 similarity cue 的使用量會受到社交風險影響。若同組參與者本來就很熟，低使用量可能代表他們不需要私下記錄或提示，而不是系統沒有用。因此正式實驗一律記錄熟悉度；若場次目標是測量「不確定是否敢公開表達」的情境，招募時應優先避免過度熟悉的朋友、同 lab、同專案或固定合作小組。

### 1.1 招募 / 排程前篩選

由研究團隊填寫，不直接在受試者信件中說明研究假設。

| 欄位 | 建議選項 / 記錄方式 |
| --- | --- |
| prior_relationship_with_group | none / met_before / classmates / labmates / project_teammates / friends / close_friends / mixed / unknown |
| known_members_count | 0 / 1 / 2 / 3 / unknown |
| collaboration_frequency_last_3_months | never / once_or_twice / monthly / weekly / multiple_times_per_week / unknown |
| same_lab_or_project | yes / no / mixed / unknown |
| recruitment_decision | less_familiar_target / acceptable_covariate / over_familiar_use_with_caveat / exclude_or_reschedule / PI_decision_needed |
| recruitment_note | 簡短描述為何仍安排或改排 |

判斷規則：

- 若三位真實參與者彼此都不熟，或只見過但沒有固定合作，標記為 `less_familiar_target` 或 `acceptable_covariate`。
- 若兩位以上是真正熟識朋友、同 lab 固定合作、同專案固定討論，且本場目標是測量 private expression，優先改排或拆組。
- 若因排程限制仍使用熟識組，必須在 session summary 標記 `over_familiar_use_with_caveat`，分析時不得把 private channel 低使用量直接解讀成系統失效。

### 1.2 Task 後小表單 / 小訪談

每個 task 後由 Observer / Interviewer 口頭詢問並代填。問題需維持中性，不提暗樁。

| 題目 | 回答格式 |
| --- | --- |
| 你在本輪 task 前，和其他組員熟悉嗎？ | 1 = 完全不熟，7 = 非常熟 |
| 你在這一輪公開提出不同意見時，覺得自在嗎？ | 1 = 非常不自在，7 = 非常自在 |
| 你有沒有因為認識或不認識組員，而改變這一輪整理想法或公開表達的方式？ | 開放回答；Observer 事後可依內容標記 affected_channel |
| 如果你沒有使用悄悄話 / idea blocks / cue，你覺得原因是功能不需要、忘記、看不懂、時間不夠，還是因為你可以直接公開說？ | 多選 + 開放回答 |
| 有沒有某個想法你先放在 private 階段，但後來因為小組氣氛或熟悉度而選擇公開或不公開？ | 開放回答 |

熟悉度 side-effect 題目的受試者可見題幹不要列出 `private_notes`、`idea_blocks`、`cue`、`chat`、`public_speech` 這類分析 tag。若受試者回答提到具體功能或行為，Observer 可在 notes 或整理表中另外標記 `affected_channel`。

### 1.3 Debrief 前個別訪談

訪談者可依觀察事件追問，但不得暗示某位成員是研究團隊安排的人。

- 剛才是否有你不太確定要不要公開講的想法？當時你怎麼判斷？
- 你和其他組員的熟悉程度，是否讓你更容易或更不容易公開反對？
- 如果同一個 task 是和完全不熟的人一起做，你覺得你會更需要先整理想法、查看可用提示，或用文字 / 口頭方式公開表達嗎？為什麼？
- 如果同一個 task 是和更熟的人一起做，你覺得你會更直接公開講嗎？為什麼？
- 剛才有沒有你覺得「大家好像已經同意了，但我其實不完全同意」的時刻？你最後怎麼處理？
- 當你看到系統提示或回顧自己的想法紀錄時，它有沒有降低你提出不同意見的心理門檻？

## 2. 分析欄位

每場 session summary 至少要記錄下列欄位，讓 familiarity 可作為 covariate 或排除條件。

| 欄位 | 建議值 |
| --- | --- |
| group_familiarity_level | none / low / medium / high / mixed / unknown |
| over_familiar_group | true / false / unknown |
| private_expression_social_risk | expected_low / expected_medium / expected_high / unknown |
| familiarity_effect_on_private_channel | reduced_need / no_obvious_effect / increased_need / mixed / unknown |
| public_disagreement_comfort_summary | 1-7 平均值 + 簡短說明 |
| analysis_caveat | 若熟悉度可能影響 private channel usage，寫明限制 |

正式分析預設不硬性排除熟識組，但必須分層或標記。若研究問題鎖定 private expression 在社交風險中的作用，`high` 或 `over_familiar_group=true` 的場次應優先視為 pilot / caveat session，除非 PI 另行決定。
