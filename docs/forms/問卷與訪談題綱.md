## 2. 表單填寫原則

- 每個 task 後都做 10 分鐘小訪談 / 小表單。
- 參與者不自行填小表單；三位 Observer / Interviewer 各自一對一口頭詢問自己負責的真實參與者並代填。
- no cue task 後填小表單 A。
- with cue task 後填小表單 B。
- 兩個 task 完成後、最後 Debrief 前填大表單；可放在個別訪談前，或訪談中由訪談者協助填。
- 暗樁不作為正式參與者填表，除非團隊另建內部紀錄表。
- 三位 Observer 各自負責自己訪談的真實參與者。
- 表單回覆必須能對應 `group_id`、`participant_id`、`task`、`condition`、`task_order`。
- 三位真實參與者個別訪談必須在最後 Debrief 前完成；訪談中不得揭露或暗示暗樁安排。

## 3. 小訪談口頭開場

Task 後小表單開場：

> 接下來會問你幾個剛才這一輪任務的問題。你不用自己填表單，我會口頭問，然後把你的回答填進表單。請只針對剛才這一輪回答。

若剛完成 no cue：

> 這一輪沒有 similarity cue，但系統仍有 idea blocks、ranking、transcript 和聊天室。請根據你剛才的實際感受回答。

若剛完成 with cue：

> 這一輪可能有 similarity cue。請根據你是否有注意到 cue、cue 是否影響你，以及 idea blocks、ranking、transcript 和聊天室的使用經驗回答。

## 4. 30 分鐘個別訪談

個別訪談是三位真實參與者分開訪談，總長約 30 分鐘，且必須在最後 Debrief 前完成。三位 Observer / Interviewer 各帶一位真實參與者到不同地方，三場訪談彼此分開、不能互相聽見。訪談應以 Google Forms 大表單、Observer notes 和具體事件為主，不要硬照一份長題綱問完。

訪談禁語：

- 不得提到「暗樁」「研究團隊安排成員」「有人是工作人員」「confederate」或任何同義說法。
- 不得用任何問題暗示暗樁存在，或詢問受試者是否察覺某位成員的身分安排。
- 若要追問某段討論，只能使用中性說法，例如「某位成員」「有人」「剛才那段討論」。
- Debrief 與暗樁說明必須等三場個別訪談都完成後才進行。

### 4.1 訪談開場

> 接下來是個別訪談，大約 30 分鐘。我們想了解你在兩個任務裡，怎麼使用 private phase、idea blocks、ranking、聊天室與 cue，以及你怎麼判斷要不要公開提出自己的想法。請根據你自己的經驗回答，不需要代表小組，也不需要給標準答案。

### 4.2 必問主題

以下是訪談者要覆蓋的主題；實際 wording 以大表單與現場事件為準。

- 兩個 task 中，哪一次比較容易提出自己的想法？為什麼？
- private phase 中產生的 idea blocks 是否幫助回顧自己的想法？
- 你覺得系統從 speech 和 text 產生的 idea blocks / notes 是否符合你剛才的想法？
- think-aloud 是否自然？會不會分心？
- public phase 中是否有想講但沒有講的內容？
- public chat 是否被視為正式公開發言管道？受試者是否曾想用私人文字訊息，但系統不提供 private chat？
- reflect phase 是否有調整個人排序？是受哪段討論影響？
- with cue task 中，same reason / different reason cue 是否被注意到？
- cue 是否增加發言信心、造成干擾，或沒有影響？
- no cue task 中，沒有提示時如何判斷切入時機？

### 4.3 Lost-at-Sea 追問

- 你在 private phase 排序時，哪些物品最不確定？
- 有沒有某個物品的理由你想講但沒有講？
- 小組 public ranking 和你的個人 ranking 差異在哪？
- reflect phase 你改了哪些物品？為什麼？
- 如果 Lost 是 with cue：cue 是否影響你提出某個物品的理由？

### 4.4 Enhance-the-Poster 追問

- Private phase 1 建立 component + action items 是否足夠清楚？
- 至少 4 個 items 是否太多、太少或剛好？
- Canva 海報與 component library 是否清楚？
- Public phase 最多選 15 個 items 是否合理？
- 你是否有想做的修改很難用 component + action 表達？
- reflect phase 你是否改變了對某些 action items 的排序？
- 如果 Poster 是 with cue：cue 是否讓你補充 same reason 或 different reason？

### 4.5 Critical incident 追問句型

- 「我注意到你在 private phase 提到 `____`，但 public phase 沒有提出。當時你在想什麼？」
- 「我注意到你看到 cue 後停了一下。你當時怎麼理解那個 cue？」
- 「我注意到你在 reflect phase 把 `____` 從第 __ 名改到第 __ 名。是哪段討論影響你？」
- 「我注意到你用 public chat 發了 `____`。為什麼選擇打字而不是口頭說？」
- 「我注意到某位成員提出 `____` 後，你的排序/發言有變化。你當時怎麼判斷？」

## 5. 表單版本管理

每次表單更新時，在 session summary 記錄：

- 小表單 A 版本 / 修改日期。
- 小表單 B 版本 / 修改日期。
- 大表單版本 / 修改日期。
- 是否有題目和 Notion brainstorm 不一致。
- 實驗當天是否有跳題、漏填或由實驗人員補填。
