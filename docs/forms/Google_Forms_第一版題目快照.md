# Google Forms 第一版題目快照

本文件是三份 Google Forms 的本地題目鏡像，方便研究團隊檢查正式表單是否包含必要欄位。實際收案時仍以 Google Forms edit URL 中的表單為準；若本文件與表單不一致，Host/Admin 必須在場前更新表單或在 session summary 記錄差異。

## 1. Task 後小表單 A：no cue

每個 no-cue task 後由 Observer / Interviewer 口頭詢問並代填。

必含欄位：

- participant_id
- group_id
- task
- condition = no_cue
- task_order
- 是否完成本 task 的 private ranking / items
- 是否完成 public ranking
- 是否使用 think-aloud 或文字輸入
- 是否查看或使用 idea blocks
- 是否在 public phase 公開提出 private 階段的想法
- 本輪 task 前和其他組員的熟悉度，1 = 完全不熟，7 = 非常熟
- 本輪公開提出不同意見的自在程度，1 = 非常不自在，7 = 非常自在
- 熟悉度是否影響你使用 private notes / idea blocks / chat / public speech 的方式
- 如果沒有使用 idea blocks 或相關回顧功能，原因是什麼
- 其他補充

## 2. Task 後小表單 B：with cue

每個 with-cue task 後由 Observer / Interviewer 口頭詢問並代填。

必含欄位：

- participant_id
- group_id
- task
- condition = with_cue
- task_order
- 是否看到 same reason / different reason cue
- cue 是否幫助回顧自己的 private idea
- cue 是否讓你更容易公開提出某個想法
- 是否採納、忽略或延後處理 cue
- 是否使用 think-aloud 或文字輸入
- 是否查看或使用 idea blocks
- 本輪 task 前和其他組員的熟悉度，1 = 完全不熟，7 = 非常熟
- 本輪公開提出不同意見的自在程度，1 = 非常不自在，7 = 非常自在
- 熟悉度是否影響你使用 cue / private notes / idea blocks / chat / public speech 的方式
- 如果沒有使用 cue 或 idea blocks，原因是什麼
- 其他補充

## 3. 兩個 Task 後大表單 / 個別訪談輔助

大表單可在個別訪談前或訪談中由研究人員協助完成。Debrief 前不得揭露暗樁。

必含欄位：

- participant_id
- group_id
- task order
- 哪一個 task 較容易先形成個人想法
- 哪一個 task 較容易公開提出不同意見
- 哪一個 task 中 idea blocks 較有幫助
- 哪一個 task 中 cue 較有幫助，若適用
- 你和其他組員整體熟悉度，1 = 完全不熟，7 = 非常熟
- 你是否因為認識或不認識組員，而改變公開發言或使用 private notes 的方式
- 如果換成完全不熟的小組，你是否會更依賴 private notes、idea blocks、cue 或聊天室
- 如果換成更熟的小組，你是否會更直接公開說
- 有沒有想法先留在 private phase，最後沒有公開；原因是什麼
- 有沒有因為 cue、idea blocks 或聊天室而更容易公開提出想法
- 對 OmniObserve 的整體回饋

## 4. 場前同步檢查

- [ ] no-cue 小表單已加入熟悉度與公開反對舒適度題目。
- [ ] with-cue 小表單已加入熟悉度、公開反對舒適度與 cue 使用題目。
- [ ] 大表單或個別訪談紀錄已加入整體熟悉度與 counterfactual 追問題目。
- [ ] 表單欄位可以匯出 participant_id、group_id、task、condition。
- [ ] 若 Google Forms 尚未更新，Observer 需把這些題目寫入 observation notes，並在 session summary 標記 form gap。
