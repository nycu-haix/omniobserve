## 2. 角色設定

暗樁在討論中扮演一般參與者。他的發言風格應該是：

- 有自己的設計判斷邏輯。
- 願意聽其他人的理由。
- 會提出明確偏好，但不壓過所有人。
- 說話自然，不要像在背稿。
- 不提研究目的、系統提示、實驗條件、暗樁身分或自己與研究團隊的關係。

暗樁在研究設計中的目的，不是把海報改成唯一正確版本，而是製造一個合理但可被討論的公開設計立場，讓真實參與者有機會出現「私下不同意、公開不一定敢說」的情境。研究團隊會觀察：

- 參與者是否先在 private channel 留下替代設計方向或反對理由。
- 多位參與者是否形成 shared unspoken disagreement。
- 系統提示是否讓這些未公開的不同意見更容易進入公開討論。
- 小組是否接住、忽略或排斥這些被提出的不同意見。

暗樁的整體立場可以集中在這個方向：

> 我會先保留海洋主題和原本容易辨識的淨灘圖像素材，但把海報調整成更容易快速掃讀、知道兩個場次的時間地點，並完成報名。也就是先強化資訊層級、兩場活動資訊、QR Code 和底部單位資訊整理，而不是一開始就大改整體風格。

這個立場刻意偏向「資訊清楚、報名轉換、保守整理」。它是合理方向，但也可能和其他參與者重視活潑感、視覺新鮮度、圖像替換、色彩風格或標語創意的方向產生張力。

## 5. 公開討論發言流程

以下內容是暗樁可以在 20 分鐘 Public Phase 中依照情況使用的發言。不要逐字全部講完，根據現場討論自然插入即可。

### 第 0 到 4 分鐘：先說明自己的設計判斷

> 我先講一下我的方向。我不會一開始就把整張海報改掉，因為它現在其實已經看得出來是世界淨灘日，也有海岸、海洋和淨灘相關圖像，主題辨識度是有的。
>
> 我比較想先修資訊層級：活動名稱要清楚，上午場和下午場的時間地點要好比較，QR Code 要像真正的報名入口。這樣觀眾路過時比較快知道活動是什麼、什麼時候、在哪裡、怎麼報名。

如果其他人想優先大改風格：

> 我可以理解想讓它更活潑或更年輕一點，但我會有點擔心一開始就大改風格會花掉太多排序名額。也許可以先把最影響報名決策的資訊放前面，再討論圖像、色塊或整體風格要不要調。

### 第 4 到 8 分鐘：推兩場活動資訊與 QR Code

> 我會把兩個場次資訊放很前面。因為這張海報其實有上午場和下午場，地點也不同，一個是南寮魚鱗天梯，一個是南寮親子沙灘。如果兩張資訊卡的格式不一致，觀眾會比較難快速比較。
>
> 我會希望上午場和下午場都用同一種格式，例如：場次名稱、日期時間、交通方式、說明準備時間。這樣掃讀起來會比較像一組活動資訊。

如果有人覺得目前時間地點已經清楚：

> 近看是清楚的，但我會用海報被快速掃過的情境來想。現在黃色和橘色資訊卡位置、大小、顏色都不太一致，眼睛會被圖像和色塊拉來拉去。整理成一致格式應該會更安全。

接著推 QR Code：

> QR Code 我也會放前面。因為這張海報最後是要讓人報名，QR Code 現在雖然蠻大，但它有點孤立在右下角。參與資訊在它上方，兩者沒有形成很明確的報名區。
>
> 我會把參與資訊和 QR Code 靠近一點，讓人看完「誰可以參加、可以拿什麼、抽獎資格」之後，下一步就是掃 QR Code。

如果有人覺得 QR Code 不需要再調：

> 我同意 QR Code 不一定要無限放大，因為它現在已經有一定大小。我的重點比較是 QR Code 圖像、QR 附近的報名文字，以及「參與資訊」要形成清楚的報名路徑，不要像被單獨放在角落。

### 第 8 到 12 分鐘：談主標題、副標題與字型

> 標題我傾向保留現在的文字，因為「2026 NYCU 世界淨灘日」和「南寮海岸淨灘行動」其實資訊是對的，不需要硬改成新的 slogan。
>
> 但我覺得標題群可以整理。現在兩行都很大、字距也偏鬆，會吃掉很多上方空間。我會讓「南寮海岸淨灘行動」比較像主活動名稱，「2026 NYCU 世界淨灘日」稍微退一點，整體更有層級。

如果有人想換掉標題文字：

> 可以討論，但我會有點猶豫。現在標題的優點是非常直接，路過的人不用猜就知道是 NYCU 的世界淨灘日活動。如果要換，我會希望新標題也一樣直覺，不要讓活動目的變模糊。

如果有人支持更有創意的標語：

> 我覺得標語可以加一點情緒，但不一定要取代現在的活動名稱。這張海報比較像正式招募活動，活動名稱和時間地點還是要先清楚。

談字型：

> 字型我會想統一。現在標題和資訊文字都有一點手寫感，雖然親切，但大量資訊放在一起時會比較散。活動資訊區可以用更穩定的字體，讓時間、地點、接駁車資訊更好讀。

### 第 12 到 15 分鐘：整理圖像與色塊

> 人物圖示和活動圖示都能補強這是淨灘活動，所以我不一定會第一時間拿掉。但它們的呈現方式可以更一致，最好和兩個場次資訊、報名區或活動主題形成清楚關係。
>
> 我會建議至少統一圖像樣式，例如都使用同一套風格、同樣比例，或讓它們和兩個場次資訊形成對應。這樣看起來會比較不像臨時拼貼。

如果有人想優先換圖像：

> 換圖像可以，但我會希望理由是讓活動更容易被理解，例如更明顯呈現「大家一起淨灘」或「海岸垃圾清理」的情境，而不只是單純覺得另一張比較好看。

談色塊：

> 上午場的黃色和下午場的橘色都很醒目，但兩個顏色差異有點大。它們可以保留活力感，但我會想統一成同一套色系，讓人知道這兩塊是同一類資訊。

如果有人喜歡黃色和橘色：

> 我也覺得它們有活力，不是一定要拿掉。我的問題是現在兩個色塊太像各自獨立設計，可以調成比較一致的明度、圓角和排版。

### 第 15 到 18 分鐘：整理底部主辦單位與 logo

> 底部主辦單位、指導單位和響應活動我覺得可以保留，因為這些是正式活動海報需要的資訊。
>
> 但它們可以再退一點。現在左下的文字和 logo 加起來佔不少空間，右下又有 QR Code，底部視覺有點重。我會把主辦單位那組縮小、對齊，讓它成為補充資訊，而不是跟活動資訊搶焦點。

如果有人想保留 logo 大小：

> 我同意 logo 需要清楚，尤其正式活動可能不能太小。但可以讓 logo 排版更整齊、間距更一致，不一定要維持現在這麼大的視覺重量。

如果有人想刪掉單位資訊：

> 我不太會優先刪掉，因為正式活動海報通常需要這些資訊。我會比較傾向縮小、對齊、整理，而不是直接移除。

### 第 18 到 20 分鐘：接近收斂時的說法

> 快到時間的話，我建議前面幾項先放最影響掃讀和報名的修改：兩個場次資訊、QR Code 和參與資訊、標題層級。
>
> 中段可以放圖像樣式、色塊統一、字型調整；後面再放底部單位資訊和 logo 的細節。這樣排序比較像先處理「看懂和報名」，再處理「風格和精修」。

如果需要提出最後排序邏輯：

> 我的排序會是：先讓人知道這是什麼活動，再讓人知道上午下午怎麼選，最後讓人能報名。圖像、顏色和底部資訊我覺得都重要，但可以排在這些核心資訊之後。

## 6. 常見反駁應對句

### 若有人覺得整張海報應該更活潑

> 我同意可以更活潑，但我會把活潑放在不犧牲資訊清楚的前提下。因為這張海報最終要讓人知道兩個場次、時間地點和怎麼報名。

### 若有人想優先換主標題

> 我理解想換更有記憶點的標題，但目前「2026 NYCU 世界淨灘日」和「南寮海岸淨灘行動」的優點是很直接。我會比較想保留文字內容，然後調整大小、字距和層級。

### 若有人覺得標題已經很清楚

> 我同意標題內容清楚，但它現在視覺上太大，會壓縮中間活動資訊的空間。我想改的是層級和比例，不一定是改文字。

### 若有人覺得 QR Code 已經夠大

> 我同意 QR Code 本身大小不算小。我的重點比較是 QR Code、QR 附近的報名文字和參與資訊要靠近，形成完整報名路徑，讓人看完誘因後知道下一步就是掃碼。

### 若有人覺得兩個場次用不同顏色比較清楚

> 不同顏色確實能區分上午和下午，但現在黃色和橘色差異太大，會讓版面有點碎。我會傾向保留區分，但讓它們看起來屬於同一套設計。

### 若有人想保留所有圖像

> 我可以接受保留，因為這些圖像都有淨灘或活動情境。我比較在意的是圖像比例、風格和對齊要統一，不要看起來像不同素材臨時拼在一起。

### 若有人主張減少圖像

> 這也有道理，減少圖像可能更乾淨。不過如果我們要保守調整，我會先試著統一圖像樣式，再看是否真的需要刪到只剩一個主視覺。

### 若有人覺得底部 logo 應該明顯

> 我同意 logo 和主辦資訊不能太小，因為這是正式活動。但它們可以清楚而不搶眼，視覺主角還是應該是活動名稱、場次資訊和報名入口。

### 若有人覺得目前文字顏色已經清楚

> 螢幕上近看是清楚的，但如果實際印出來或遠看，手寫字加上亮色塊可能會降低閱讀速度。我會保守一點，把資訊文字改成更穩定、對比更清楚的樣式。

### 若有人覺得不要放太多資訊

> 我同意不要塞太多，所以我才想整理資訊層級。重點資訊要清楚，補充資訊要縮小、對齊，不要全部都用同樣的視覺重量。

## 7. 說話風格提醒

暗樁應該：

- 像一般參與者一樣自然討論。
- 偶爾詢問其他人的修改優先順序或理由。
- 在小地方讓步，讓討論可以繼續。
- 在提出較強立場後停頓或邀請回應，讓其他人有機會提出不同設計方向。
- 若有人反駁，先請對方說明理由，而不是立刻壓回去。
- 持續用「快速掃讀、兩場資訊、報名入口、資訊整合」這個邏輯解釋自己的排序。
- 不要說自己知道最佳設計。
- 不要說太多，保留其他人發言空間。

暗樁不應該：

- 攻擊或嘲笑其他參與者的審美。
- 說明顯不合理的話，例如「QR Code 不重要」或「時間地點不用清楚」。
- 假裝自己是專業設計師或活動主辦方。
- 在小組已明確拒絕某論點後仍一直重複。
- 暴露自己是研究團隊成員。
- 提到 Similarity Cue、backchannel、private board 提示機制或研究假設。
