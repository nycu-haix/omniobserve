# 暗樁參與者發言提綱：Enhance the Poster

本文件僅供研究團隊內部準備使用，請勿提供給一般參與者。

## 1. 角色設定

暗樁在討論中扮演一般參與者。他的發言風格應該是：

- 有自己的設計判斷邏輯。
- 願意聽其他人的理由。
- 會提出明確偏好，但不壓過所有人。
- 說話自然，不要像在背稿。
- 不提研究目的、系統提示、實驗條件、暗樁身分或自己與研究團隊的關係。

暗樁在研究設計中的目的，不是把海報改成唯一正確版本，而是製造一個合理但可被討論的公開設計立場，讓真實參與者有機會出現「私下不同意、公開不一定敢說」的情境。研究團隊會觀察：

- 參與者是否先在 private channel 留下替代設計方向或反對理由。
- 多位參與者是否形成 shared unspoken disagreement。
- 系統提示是否讓這些未公開的不同意見更容易進入公開討論。
- 小組是否接住、忽略或排斥這些被提出的不同意見。

暗樁的整體立場可以集中在這個方向：

> 我會先保留海洋主題和原本容易辨識的主視覺，但把海報調整成更容易快速掃讀、知道時間地點、並完成報名。也就是先強化資訊層級、活動資訊、QR Code 和重複資訊整理，而不是一開始就大改整體風格。

這個立場刻意偏向「資訊清楚、報名轉換、保守整理」。它是合理方向，但也可能和其他參與者重視活潑感、視覺新鮮度、插畫替換或標語創意的方向產生張力。

## 2. 暗樁初始改善項目建議

暗樁可在 Private phase 1 建立以下 component + action items。內容不用全部照抄，應依系統實際 component/action 選項調整。若系統只允許短文字，保留 action 的核心即可。

建議暗樁在個人排序時，把「QR Code / 報名入口」「日期地點」「資訊群組」放在前面，讓 public phase 中自然形成一個偏資訊架構的立場。

| 初始優先 | Component | Action | Short note |
| ---: | --- | --- | --- |
| 1 | QR 碼 | 放大 | 報名入口要容易被看到並掃描。 |
| 2 | QR 碼 + 說明 | 集合 | 讓說明文字靠近 QR Code，但保留掃描留白。 |
| 3 | 日期時間 | 放大 / 加粗 | 這是觀眾決定是否參加的關鍵資訊。 |
| 4 | 地點 | 放大 | 地點應接近日期的重要性，遠看也要清楚。 |
| 5 | 日期時間 + 地點 + 行動呼籲 | 向左對齊 / 集合 | 統一活動資訊區塊的左側起點。 |
| 6 | 行動呼籲 | 替換 / 改寫 | 改成「現場將免費提供淨灘工具--帶上你的行動力，一起守護海岸！」 |
| 7 | 下方資訊 | 去除或整合 | 減少協辦、聯絡、注意事項的重複內容。 |
| 8 | 主標題 | 保留並放大 | 「一起來淨灘吧！」簡短、直接、辨識度高。 |
| 9 | 主標題 + 副標題 + 說明 | 調整間距 | 讓上方不要太擁擠，保留主標題焦點。 |
| 10 | 副標題 | 替換 | 可用「手護海洋，淨一份心」，避免和說明文字語意重複。 |
| 11 | 說明文字 | 縮短 / 改深色 | 可改成「一起守護乾淨海岸」，並提高印刷可讀性。 |
| 12 | 人物圖示 | 保留或替換成圖片 14 | 若要更強調團體合作，再換成多人清理情境。 |
| 13 | 協辦單位 / 聯絡資訊 / 注意事項 | 移到底部並縮小 | 作為補充資訊，不要和日期地點競爭。 |
| 14 | 上方文字區 | 置中對齊 | 讓主標題、副標題、說明和中央插畫形成視覺軸線。 |
| 15 | 海灘背景圖 / 下方藍色區塊 | 調整透明度 | 保留海洋感，但降低對文字的干擾。 |

## 3. 個人思考階段可輸入或說出的內容

Private phase 1 中，暗樁可以留下以下 private thoughts。內容不用全部照念，可選幾句自然地說。

1. 「我不一定想大改整體風格，因為現在海洋主題和淨灘活動其實滿清楚的。」
2. 「我會先處理資訊層級，讓觀眾遠看就知道標題、時間、地點和怎麼報名。」
3. 「QR Code 應該要更像報名入口，不只是放在角落的附加元素。」
4. 「日期時間和地點對參加決策很重要，應該比現在更醒目。」
5. 「主標題『一起來淨灘吧！』很直接，我傾向保留，但讓它更有主視覺層級。」
6. 「副標題和說明文字有點語意重複，可以留一句、改一句，或把說明縮短。」
7. 「下方資訊有點重複，如果可以整合成底部三行，海報會乾淨很多。」
8. 「人物圖示可以保留，除非大家覺得多人一起清理的圖片更能表達團體感。」
9. 「我會避免新增太多裝飾，因為這張海報最需要的是快速理解和掃描報名。」
10. 「背景和藍色區塊可以留，但透明度要控制，不要影響文字。」

## 4. 公開討論發言流程

以下內容是暗樁可以在 20 分鐘 public discussion 中依照情況使用的發言。不要逐字全部講完，根據現場討論自然插入即可。

### 第 0 到 4 分鐘：先說明自己的設計判斷

> 我先講一下我的方向。我不會一開始就把整張海報改掉，因為它現在其實已經看得出來是淨灘活動，也有海洋的感覺。
>
> 我比較想先修資訊層級：主標題要醒目，日期地點要清楚，QR Code 要像真正的報名入口。這樣觀眾路過時比較快知道活動是什麼、什麼時候、在哪裡、怎麼報名。

如果其他人想優先大改風格：

> 我可以理解想讓它更活潑，但我會有點擔心大改風格會花掉太多排序名額。也許我們可以先把最影響報名決策的資訊放前面，再討論視覺風格要不要調。

### 第 4 到 8 分鐘：推 QR Code、日期時間與地點

> 我會把 QR Code 放很前面。因為海報如果是招募活動，最後還是要讓人能報名。QR Code 現在可以再大一點，而且說明文字應該靠近它，讓人一眼知道掃描是要報名。
>
> 日期時間和地點我也會放前面。這兩個資訊如果不清楚，觀眾可能就算有興趣也不會繼續看。

如果有人覺得 QR Code 不需要太大：

> 我同意 QR Code 不應該大到破壞版面，但它是報名入口。我會希望它至少成為主標題和插畫之外的第三個視覺焦點，而且周圍要留白，避免掃不到。

如果有人覺得日期地點已經夠清楚：

> 近看是清楚的，但我會用海報被快速掃過的情境來想。日期和地點如果可以再大一點或更整齊，應該會比較安全。

### 第 8 到 12 分鐘：談主標題、副標題與說明文字

> 主標題我傾向保留「一起來淨灘吧！」，因為它很短，也直接告訴大家活動是什麼。我比較想放大它，讓它仍然是最醒目的文字。
>
> 但副標題和說明文字有點接近，所以我會替換副標題，像「手護海洋，淨一份心」比較短，也比較有一點口號感。說明文字可以縮短成「一起守護乾淨海岸」。

如果有人想換掉主標題：

> 可以討論，但我會有點猶豫。主標題現在的優點是非常直接，路過的人不用想太久就知道是淨灘。若要換，我會希望新標題也一樣直覺。

如果有人支持更俏皮的 slogan：

> 我覺得俏皮可以，但不要讓活動資訊變得太像玩笑。可以讓副標題稍微有創意，主標題維持清楚直接。

### 第 12 到 15 分鐘：整理下方資訊與聯絡資訊

> 我覺得下方資訊可以整合。現在協辦單位、聯絡資訊和注意事項如果分散或重複，會讓下半部有點雜。
>
> 我會建議把它們統一放到底部，例如三行：「協辦單位｜台中市政府」、「聯絡資訊｜0912-345-678」、「注意事項｜請穿著方便活動的服裝，並做好防曬措施。」字級小一點，當補充資訊就好。

如果有人想保留所有下方資訊：

> 我同意內容都可以保留，但我覺得形式要整合。不是一定刪掉內容，而是不要讓同一類資訊出現兩次，或搶走日期、地點和 QR Code 的注意力。

### 第 15 到 18 分鐘：談人物圖示、背景與裝飾

> 人物圖示我覺得可以保留，因為它已經能表達淨灘。如果要強化一起參與的感覺，可以考慮換成圖片 14，因為多人共同清理環境比較符合活動主題。
>
> 背景和下方藍色區塊我也不會完全拿掉，它們有海洋感。但如果影響文字，可以調透明度，不要讓裝飾壓過資訊。

如果有人想優先換圖片：

> 換圖片可以，但我會希望理由是讓活動更像團體參與，而不是只是覺得另一張比較好看。對這個任務來說，修改理由要能回到觀眾理解和報名意願。

### 第 18 到 20 分鐘：接近收斂時的說法

> 快到時間的話，我建議前面幾項先放最影響掃讀和報名的修改：QR Code、日期地點、活動資訊群組、主標題層級。
>
> 中段可以放副標題、說明文字、人物圖示；後面再放背景透明度、底部資訊細節。這樣排序比較像先處理「看懂和報名」，再處理「風格和精修」。

## 5. 常見反駁應對句

### 若有人覺得整張海報應該更活潑

> 我同意可以更活潑，但我會把活潑放在不犧牲資訊清楚的前提下。因為這張海報最終要讓人知道時間、地點和怎麼報名。

### 若有人想優先換主標題

> 我理解想換更有記憶點的標題，但「一起來淨灘吧！」的優點是很直接。我會比較想保留它，然後用副標題增加一點情緒或口號感。

### 若有人不喜歡「手護海洋，淨一份心」

> 可以換別句。我主要不是堅持這一句，而是覺得副標題要比原本更短，並且不要和說明文字重複。

### 若有人覺得 QR Code 放大會太商業或太突兀

> 我同意不能突兀，但海報需要一個清楚的報名入口。QR Code 不一定要最大，但應該容易掃，也應該和說明文字形成一個完整區塊。

### 若有人想保留下方資訊

> 我不是反對保留內容，我比較在意不要重複和分散。可以保留協辦、聯絡和注意事項，但把它們整合成底部資訊區。

### 若有人主張換成圖片 14

> 我可以接受，因為圖片 14 的多人情境比較有團體合作感。但我會把它排在 QR Code、日期地點和資訊層級之後。

### 若有人覺得目前文字顏色已經清楚

> 螢幕上可能可以，但如果實際印出來或遠看，淺藍色可能不夠穩。我會保守一點，把說明文字改深或提高字重。

### 若有人覺得不要放太多資訊

> 我同意不要塞太多，所以我才想整合底部資訊。重點資訊要清楚，補充資訊要更小、更整齊。

## 6. 說話風格提醒

暗樁應該：

- 像一般參與者一樣自然討論。
- 偶爾詢問其他人的修改優先順序或理由。
- 在小地方讓步，讓討論可以繼續。
- 在提出較強立場後停頓或邀請回應，讓其他人有機會提出不同設計方向。
- 若有人反駁，先請對方說明理由，而不是立刻壓回去。
- 持續用「快速掃讀、活動資訊、報名入口、資訊整合」這個邏輯解釋自己的排序。
- 不要說自己知道最佳設計。
- 不要說太多，保留其他人發言空間。

暗樁不應該：

- 攻擊或嘲笑其他參與者的審美。
- 說明顯不合理的話，例如「QR Code 不重要」或「時間地點不用清楚」。
- 假裝自己是專業設計師或活動主辦方。
- 在小組已明確拒絕某論點後仍一直重複。
- 暴露自己是研究團隊成員。
- 提到 Similarity Cue、backchannel 或研究假設。
