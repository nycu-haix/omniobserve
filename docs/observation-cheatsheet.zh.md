# OmniObserve 單人觀察 Cheatsheet

本文件給單一觀察者使用。觀察者的核心任務是記錄受試者在 **Private idea → Public discussion → Similarity Cue → 是否 engage** 之間的行為轉換，並避免在實驗過程中主動引導受試者。

## Page 1. 實驗期間

### 觀察者原則

- 只回答 technical issue，例如頁面壞掉、按鈕在哪裡、麥克風是否有開。
- 不主動鼓勵受試者發言，不說「你可以講啊」、「有想法就說出來」。
- 不提醒受試者把 private idea 拿去 public 講，除非這是主持人在統一講稿中已經說明的規則。
- 不坐在自己認識的受試者旁邊。
- 不長時間盯著受試者看；可假裝忙於記錄或看資料，以降低壓力。
- 若受試者問內容策略，例如「我該不該講？」、「這個物品該排哪裡？」請不要給建議，只能回到任務規則。

### 核心問題

private channel / Idea Box 裡的疑問、不同意見、支持訊號或新想法，有沒有被帶進 public discussion？如果沒有，是因為不敢講、不知道何時切入、覺得不重要、忘記、還是系統 / UI 造成阻礙？

| 對應 | 觀察重點 |
| --- | --- |
| H1 | 看到 shared unspoken thought / cue 後，受試者是否更敢表達不同意見？ |
| H2 | cue 或 shared support 後，受試者是否提出更完整的理由、比較、替代方案？ |
| H3 | cue 是否被受試者接受，還是被視為干擾？ |
| H4 | 受試者是否願意把還不確定、不同意或未成熟的想法放進 private channel / Idea Box？ |
| D1 | private input 是否輕量，不會搶走主討論注意力？ |
| D2 | cue 是否只在重要時刻出現，沒有打斷討論？ |
| D3 | 受試者是否保有「要不要說、何時說、怎麼說」的控制感？ |
| D4 | 受試者是否清楚 private / public 邊界，不會擔心 private 內容被公開？ |
| D5 | 受試者是否知道誰正在 public speaking 或移動 public ranking？ |

### 各階段觀察重點

#### 階段 1：Private idea 產生（8 分鐘）

- 受試者是否完成初始個人排序？
- 受試者是否至少留下 2 個 private thoughts / idea blocks？
- 他留下的是關鍵字、直覺、完整理由、疑問、反對意見，還是排序判斷？
- 他是用語音、打字，還是完全不用？
- 他是否覺得用語音記錄 private thoughts 會分心？
- 他是否把 private channel 當筆記工具，而不是發言前緩衝？
- 他是否長時間 muted，疑似忘記切到 private mode？
- 他是否擔心 private 內容被其他人看到？

#### 階段 2：Public discussion（20 分鐘）

- 每位受試者是否至少公開發言 2 次？
- 受試者是否把 private idea 帶進 public discussion？
- 若沒有帶出來，是因為不敢講，還是不知道何時切入？
- 其他人熱烈討論時，受試者是否出現想講但停住、張口又停、滑鼠移動到麥克風但沒開等行為？
- 受試者是否根據 public discussion 修改 public ranking 或影響他人修改？
- Public ranking 被移動時，其他人是否注意到是誰在移動？
- 排序 item 是否突然變動，造成討論和操作脫節？

#### 實驗組：Similarity Cue 觀察

- Cue 出現時，當下正在討論什麼 item？
- Cue 是否和目前討論時機相符，還是太早 / 太晚？
- Cue 是否一次出現太多，造成資訊過載？
- Cue 是否太快消失？
- Cue 是 Same Reason（綠色）還是 Different Reason（黃色）？受試者看得懂嗎？
- 受試者看到 Cue 後是：
  - 馬上發言
  - 停頓後發言
  - 看了但忽略
  - 顯得困惑
  - 被打斷或分心
  - 變得更不敢講
- 若沒有 Cue，受試者是否可能解讀成「沒有人跟我想法相近」？

#### 階段 3：Final private ranking（2 分鐘）

- 受試者是否有修改自己的 private ranking？
- 修改是否對應剛剛 public discussion 中被討論過的 item？
- 修改是否可能和某個 Cue、某個人的發言、或 public ranking 操作有關？
- 受試者是否想繼續討論但被切到 final private ranking？

### 優先記錄的事件

- private 有想法，但 public 沒有說。
- private 想法和 public 討論方向不一致。
- 受試者想講但停住，或不知道何時切入。
- 受試者公開提出不同意見。
- 受試者提出更完整、有理由的貢獻。
- cue 出現後，受試者發言、忽略、困惑或覺得被打斷。
- cue 促成的想法被小組接住、忽略或否定。
- 受試者擔心 private thoughts 被看到。
- 受試者混淆右側逐字稿、聊天室、private / public 顯示區。
- 受試者覺得 private input 麻煩或分心。
- 受試者偏好打字而不是語音 private channel。
- 受試者忘記切換麥克風模式，長時間停在 muted。
- public ranking 被移動，但其他人不知道是誰移動。
- 受試者注意到排序差異標示、紅綠 / 黃綠提示，但不理解意思。
- 受試者在 final private ranking 階段修改了原本排序。

### 建議記錄格式

| 時間 | 階段 | 受試者 | 事件 | 相關 item | private / public / cue | 觀察者推測 | 後訪要追問 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 例：12:35 | Public | P2 | 看 Cue 後停頓但沒有發言 | 蚊帳 | cue | 可能不知道何時切入 | 問他看到 Cue 時在想什麼 |

## Page 2. 半結構式訪談

開場：

> 我想請你回想剛剛討論時，哪些想法你選擇先留在 private channel / Idea Box，哪些想法後來帶進公開討論。沒有標準答案，我們主要想了解你當下怎麼判斷要不要說、什麼時候說，以及系統或介面有沒有影響你。

### 必問題，所有人

1. 剛剛有沒有任何想法，你有想到或記在 private channel / Idea Box，但最後沒有公開說出來？為什麼？
2. 那個時候其他人在討論什麼？你沒有講是因為不確定內容、不太敢講、覺得不重要，還是不知道什麼時候切入？
3. 有沒有任何 private channel / Idea Box 裡的想法，後來被你帶進公開討論？你為什麼決定說出來？
4. 你在 private channel / Idea Box 通常會留下什麼？例如關鍵字、疑問、不同意見、支持理由、還沒整理好的想法、完整理由。
5. 知道 private thoughts 不會自動公開，會不會讓你更願意先記下想法？為什麼？
6. 使用 private channel / Idea Box 會不會增加負擔，或讓你分心於主要討論？
7. 你比較偏好用語音還是打字留下 private thoughts？為什麼？
8. 如果知道「你不是唯一有這個方向想法的人」，會不會讓你更有信心提出不同意見？為什麼？
9. 你有沒有擔心自己 private 的內容被公開，或不確定哪些內容別人看得到？
10. 你覺得 public ranking 被移動時，你知道是誰在移動、為什麼移動嗎？這會影響你討論嗎？
11. 你覺得排序差異標示、顏色、數字或提示容易理解嗎？有沒有哪裡讓你誤解？
12. 最後 2 分鐘調整自己的 private ranking 時，你有改排序嗎？如果有，是什麼讓你改？

### 實驗組加問

13. 你有注意到系統提示嗎？你覺得它是在幫助你、打斷你，還是沒有影響？
14. 有沒有任何提示讓你更想把不同意見、疑問或不確定想法說出來？
15. 有沒有任何提示你選擇忽略？為什麼？
16. 提示有沒有讓你提出更完整的理由、比較或替代方案？
17. 提示出現的時間點如何？太早、剛好、太晚，還是原本話題已經過去了？
18. 如果你沒有看到提示，你會不會解讀成沒有人跟你想法相近？
19. Same Reason / Different Reason 的提示差異清楚嗎？綠色 / 黃色對你來說代表什麼？
20. 如果一次出現多個提示，你會知道先看哪一個或先處理哪一個嗎？

### UI / UX 加問（視情況）

- 你覺得右側欄位資訊會不會太多？有沒有想把它收起來？
- 你有沒有希望 transcript 和 chat 分開？
- 你有沒有希望可以手動修改語音轉文字結果？
- 你是否清楚目前自己是 muted、public speaking 還是 private speaking？
- 你有沒有忘記切換麥克風模式？
- 你覺得 GC / 頭像 / 發話者顯示有幫助嗎？還是會想收起來？
- 你有沒有希望文字或任務說明可以放大？

### Critical Incident 追問句型

> 我注意到你在 private channel / Idea Box 提到「____」，但公開討論時沒有提出。你當時為什麼沒有講？

> 我注意到當時大家正在討論「____」，你看起來想講但停住了。你當時是在想內容，還是在等切入時機？

> 我注意到你看到提示後有停頓 / 發言 / 忽略。你當時在想什麼？

> 我注意到你後來提出「____」。是什麼讓你決定把它說出來？

> 我注意到小組有 / 沒有接著討論這個想法。這會影響你之後願不願意再提出想法嗎？

> 我注意到 public ranking 在「____」被移動時，你有看一下 / 沒有反應。你當時知道誰在移動嗎？

> 我注意到最後 2 分鐘你把「____」從第 __ 名改到第 __ 名。這和剛剛哪段討論或提示有關嗎？
