# OmniObserve 單人觀察 Cheatsheet

## Page 1. 實驗期間

看這位受試者的 private thoughts 如何進入或沒有進入 public discussion。

1. **Private 有，public 沒有**：受試者私下留下疑問、反對、理由或新想法，但公開討論沒有提出。
2. **Private 和 public 不一致**：受試者的 private thought / individual ranking 和小組公開討論方向不同。
3. **Cue 後反應**：受試者看到 cue 後是更敢說、忽略、困惑、覺得被打斷，還是小組有接住這個想法。

### 對應老師 H1-H4 / D1-D3

| 代碼 | 實驗中看什麼 |
| --- | --- |
| H1 | shared unspoken thoughts 是否讓受試者更有信心提出不同意見。 |
| H2 | shared support 是否讓受試者提出更有品質的貢獻，例如理由、比較、替代方案、整合。 |
| H3 | cue 是否因為有 prior alignment 而比較容易被接受，不像任意打斷。 |
| H4 | 受試者是否因為可控制是否公開，而更願意把想法先放進 private channel。 |
| D1 | backchannel 是否沒有增加太多認知負擔。 |
| D2 | cue 是否沒有干擾主要討論流程。 |
| D3 | 受試者是否保有是否分享、何時分享、怎麼分享的自主性。 |

### 介面熟悉階段

注意：

- 是否分得清楚 public mic 和 private mic / private text。
- 是否知道 group chat 是公開頻道。
- 是否知道 private thoughts 不會自動公開。
- 是否會操作 ranking。
- 是否出現隱私疑慮或操作困難。

### 個人思考階段

注意：

- 有沒有使用 private input：語音、文字、兩者都有、完全不用。
- private thoughts 是什麼類型：疑問、反對、支持理由、不確定、新想法、排序理由。
- 是否只做 ranking，沒有留下任何理由。
- 是否看起來想講但沒有使用 private channel。
- 是否因為操作麻煩、怕被看到、不知道要說什麼而不用。

### 小組討論階段

注意：

- private idea 是否被帶入公開討論。
- private idea 是否一直沒有公開提出。
- 是否出現 private-public mismatch。
- 是否提出 disagreement、challenge、保留意見。
- 是否看起來想發言但停住，例如手移到 mic、開口又停、打字又刪。
- 是否因為他人強勢論點而改變排序。
- 是否用 group chat 取代語音發言。
- 公開貢獻是否有品質：有理由、比較、假設、替代方案、整合他人觀點。

### 實驗組 Cue 出現時

注意：

- cue 出現時間。
- 受試者是否明顯看到 cue。
- cue 後 30 秒內有沒有發言、打字、改 ranking。
- cue 是否讓受試者更敢提出不同意見。
- cue 是否讓受試者困惑、忽略或覺得被打斷。
- cue 促成的想法是否被小組接住、討論、整合、忽略或否定。

### 最值得記的 Critical Incidents

優先記這些：

- private 有想法但 public 沒有說。
- private 和 public 討論方向明顯不同。
- 看到 cue 後停頓、發言、忽略或困惑。
- 想發言但停住。
- 因強勢意見改 ranking。
- 明確提出不同意見。
- 提出高品質理由或替代方案。
- 擔心 private thoughts 被看到。
- 不理解 mic、cue、ranking 或 chat。

## Page 2. 半結構式訪談問什麼

訪談目標：讓受試者回想他如何決定「哪些想法留在 private channel，哪些想法帶進 public discussion」。不要一開始說「我們在測 cue 有沒有讓你更敢說」。

### 開場

> 接下來想請你回想剛剛使用系統和小組討論的經驗。沒有標準答案，我們主要想了解你當下怎麼想、怎麼決定要不要說出某些想法。

### 暖身與整體經驗

1. 你可以簡單描述剛剛整個小組討論過程嗎？
2. 你覺得自己在討論中表達想法容易嗎？為什麼？
3. 有沒有任何時候你有想法，但最後沒有說出來？

### Private Channel 使用經驗

4. 個人思考階段，你怎麼決定自己的排序和理由？
5. 你有使用 private mic 或 private text 嗎？為什麼有或沒有？
6. 你通常把哪些內容放在 private channel？例如疑問、反對、支持理由、不確定的想法。
7. 知道 private thoughts 不會自動公開，會不會影響你願不願意先記下想法？
8. 使用 private channel 會不會增加你的負擔，或讓你分心於主要討論？

### Private-to-Public 轉換

9. 有沒有任何 private channel 中的想法，後來被你帶到公開討論？
10. 你怎麼決定那個想法可以公開說出來？
11. 有沒有任何 private 想法最後沒有提出？為什麼？
12. 沒有提出的原因比較接近哪一種：不確定、怕打斷、別人已經講了、覺得自己是少數、時間不夠，還是其他？
13. 有沒有哪一刻你其實不同意別人的說法，但沒有馬上講？當時在考慮什麼？

### Shared Unspoken Thoughts and Confidence

14. 如果系統讓你知道「你不是唯一有這個方向想法的人」，會不會讓你比較有信心提出不同意見？為什麼？
15. 剛剛有沒有哪個時刻，你因為覺得可能有人支持你或有類似想法，而比較願意發言？
16. 有沒有哪個時刻，即使知道可能有人有類似想法，你仍然不想講？為什麼？

### Cue 相關問題，實驗組才問

17. 你有注意到系統提示嗎？第一次看到時，以為它代表什麼？
18. 你覺得提示是在幫助你、打斷你，還是沒有太大影響？
19. 有沒有任何提示讓你更想把不同意見、疑問或不確定想法說出來？
20. 有沒有任何提示你選擇忽略？為什麼？
21. 你覺得提示出現的時機合理嗎？有沒有太早、太晚或太多？
22. 提示有沒有讓你提出更完整的理由、比較或替代方案？
23. 你會不會擔心系統提示讓別人猜到你的 private thoughts？

### Critical Incident 追問句型

> 我注意到你在第 __ 分鐘時看了一下提示，但後來沒有發言。你當時在想什麼？

> 我注意到你在 private channel 提到「____」，但公開討論時沒有提出。你還記得當時為什麼沒有講嗎？

> 我注意到某位參與者提出「____」後，你改了排序。是什麼讓你改變？

> 我注意到你後來提出了「____」，而這跟你前面 private 記錄的內容有關。你當時為什麼決定把它說出來？

> 我注意到提示出現後，小組有/沒有接著討論這個想法。你覺得那個提示對你或小組有影響嗎？

### 收尾

24. 如果之後再用這個系統討論，你希望 private channel 或提示怎麼改？
25. 你覺得這個系統比較適合幫助哪一種想法被提出：疑問、反對、支持理由、還是新點子？為什麼？
26. 還有沒有任何你剛剛想說、但我沒有問到的地方？
