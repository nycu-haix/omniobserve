# OmniObserve 單人觀察 Cheatsheet

## Page 1. 實驗期間

### 核心問題

private channel 裡的疑問、不同意見、支持訊號或新想法，有沒有被帶進 public discussion？

| 對應 | 觀察重點 |
| --- | --- |
| H1 | 看到 shared unspoken thought / cue 後，受試者是否更敢表達不同意見？ |
| H2 | cue 或 shared support 後，受試者是否提出更完整的理由、比較、替代方案？ |
| H3 | cue 是否被受試者接受，還是被視為干擾？ |
| H4 | 受試者是否願意把還不確定、不同意或未成熟的想法放進 private channel？ |
| D1 | private input 是否輕量，不會搶走主討論注意力？ |
| D2 | cue 是否只在重要時刻出現，沒有打斷討論？ |
| D3 | 受試者是否保有「要不要說、何時說、怎麼說」的控制感？ |

### 優先記錄的事件

- private 有想法，但 public 沒有說。
- private 想法和 public 討論方向不一致。
- 受試者想講但停住。
- 受試者公開提出不同意見。
- 受試者提出更完整、有理由的貢獻。
- cue 出現後，受試者發言、忽略、困惑或覺得被打斷。
- cue 促成的想法被小組接住、忽略或否定。
- 受試者擔心 private thoughts 被看到。
- 受試者覺得 private input 麻煩或分心。

## Page 2. 半結構式訪談

開場：

> 我想請你回想剛剛討論時，哪些想法你選擇先留在 private channel，哪些想法後來帶進公開討論。沒有標準答案，我們主要想了解你當下怎麼判斷。

### 必問題，所有人

1. 剛剛有沒有任何想法，你有想到或記在 private channel，但最後沒有公開說出來？為什麼？
2. 有沒有任何 private channel 裡的想法，後來被你帶進公開討論？你為什麼決定說出來？
3. 你在 private channel 通常會留下什麼？例如疑問、不同意見、支持理由、還沒整理好的想法。
4. 知道 private thoughts 不會自動公開，會不會讓你更願意先記下想法？為什麼？
5. 使用 private channel 會不會增加負擔，或讓你分心於主要討論？
6. 如果知道「你不是唯一有這個方向想法的人」，會不會讓你更有信心提出不同意見？為什麼？

### 實驗組加問

7. 你有注意到系統提示嗎？你覺得它是在幫助你、打斷你，還是沒有影響？
8. 有沒有任何提示讓你更想把不同意見、疑問或不確定想法說出來？
9. 有沒有任何提示你選擇忽略？為什麼？
10. 提示有沒有讓你提出更完整的理由、比較或替代方案？

### Critical Incident 追問句型

> 我注意到你在 private channel 提到「____」，但公開討論時沒有提出。你當時為什麼沒有講？

> 我注意到你看到提示後有停頓 / 發言 / 忽略。你當時在想什麼？

> 我注意到你後來提出「____」。是什麼讓你決定把它說出來？

> 我注意到小組有 / 沒有接著討論這個想法。這會影響你之後願不願意再提出想法嗎？
