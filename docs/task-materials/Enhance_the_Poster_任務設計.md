# Enhance the Poster 任務設計

本文件定義正式版本的 Enhance-the-Poster task。正式材料會使用 Canva 觀看海報與可改變 component library：

<https://www.canva.com/design/DAHLWDPZD4Q/6kUgxerChUCU51U2tsUAQA/edit>

Component/action taxonomy 目前採用本文件版本作為正式草案。正式執行前若 Canva 或 component library 有更新，需在 session summary 記錄使用版本；若更新版和本文件衝突，先標註差異並請 PI/團隊決定，不直接覆蓋正式 procedure。

## 1. 任務目標

參與者要針對 Canva 中的海報與 component library 提出 component + action items，並在小組中排序最多 15 個最重要 items。任務不是讓參與者實際美編，也不是測試設計能力；重點是觀察：

- 個人階段形成的修改意圖是否進入公開討論。
- 多個合理設計方向如何競爭。
- 當主流方向形成時，少數但合理的觀點是否被表達。
- OmniObserve cue 是否協助參與者把 private 想法帶入 public discussion。
- Public phase 是否能促成每位參與者至少兩次公開發言。

## 2. Phase

| Phase | 時間 | 任務 |
| --- | ---: | --- |
| Private phase 1 | 5 分鐘 | 每人建立至少 4 個 component + action items，最多無上限 |
| Private phase 2 | 7 分鐘 | 對整理後的 items 做個人排序；進入 Public phase 前務必先完成自己的前 15 項排序 |
| Public phase | 20 分鐘 | 小組討論，最多選 15 個 items，從最重要到最不重要排序；每位參與者至少公開發言兩次 |
| Reflect phase | 2 分鐘 | 不再公開討論，各自調整個人排序 |

## 3. 任務材料結構

正式材料以 Canva 為主。參與者使用 Canva 看海報與 component library，但不直接修改海報。系統內只記錄文字形式 component + action item。

每位參與者收到的 Canva 連結中應包含：

- 初版淨灘活動招募海報。
- 本任務的任務說明與 phase 需求。
- Reviewer feedback 摘要，若本輪材料版本有提供；講稿不逐字朗讀完整 reviewer 原文。
- 圖床 / image library，提供可替換主視覺、背景或輔助圖片。
- 文字與 slogan library，提供可替換的主標題、副標題、說明文字、CTA、QR caption 等候選文字。
- Component/action library，讓參與者能對照系統中的 component 與改善動作建立 task items。

正式執行前，Host 需在 session summary 記錄 `review_feedback_available=yes/no` 與 Canva material version。若本輪沒有提供 reviewer feedback，小表單與訪談中所有 reviewer feedback 題目需依 [功能使用與 Skip Logic](../forms/功能使用與Skip_Logic.md) 標記 `N/A_not_available`。

### 材料 1: 任務說明

必含：

- 活動背景。
- 目標受眾。
- 海報要達成的目的。
- 限制條件。
- 操作方式：只用文字寫 component + action item，不直接修改海報。
- 禁止事項：不要上網搜尋素材，不新增未提供圖片，除非題目允許自創文字。

正式任務說明：

> 你們已完成一張淨灘活動招募海報的初稿。為了進一步提升海報品質，請綜合目前的海報內容、任務需求與提供的素材，討論哪些部分需要調整，讓整體視覺呈現更加完整以及和諧。若這一輪 Canva 材料有提供 reviewer feedback，請把它當作其中一項參考；若沒有提供，請直接以海報內容、任務需求、圖床、文字 / slogan library 與自己的設計判斷建立改善項目。
>
> Private Phase 1 請從預設的海報元件與改善動作中建立至少 4 個具體 task items，沒有數量上限，並將你認為最應該優先改善的項目排在前面。建立 task item 時請考慮觀眾能否快速理解活動內容、是否願意報名、時間地點是否清楚、QR Code 是否容易掃描、視覺層次是否有效，以及資訊是否足以讓人安心參與。
>
> Private Phase 2 有 7 分鐘，只需要排序前 15 個最重要的改善項目；第 16 個之後代表不會改動。這一階段請務必先完成自己的個人排序，Public Phase 會以這份排序作為初始個人清單。
>
> Public Phase 的排序動作與 Private Phase 2 相同，但在此階段可以開啟公開麥克風和其他人進行公開討論，並且請每位參與者在此階段進行至少兩次公開發言。

### 材料 2: 初始海報

要求：

- 不要只放一個明顯錯誤。
- 同時包含 layout、hierarchy、readability、image choice、tone、call to action、information grouping 等多個可討論方向。
- 初始海報中所有重要區塊要有 component ID。

### 材料 3: Reviewer feedback 摘要（可選）

Canva 可以放 reviewer feedback 原始內容或摘要，但這是材料版本的一部分，不得在沒有提供 feedback 的場次中由講稿承諾存在。正式說明只提醒參與者：若材料包含 reviewer feedback，它主要集中在資訊清楚度、文字內容與視覺呈現。

建議在 Canva 中用摘要卡呈現：

| Feedback ID | 摘要方向 | 回饋重點 |
| --- | --- | --- |
| F1 | 標題層級 | 主視覺與主標題清楚，但副標題和說明語意接近，可減少重複並拉開上方文字層級。 |
| F2 | 資訊可讀性 | 底部協辦、聯絡資訊與注意事項偏小；必要資訊可放大、增加行距或縮短次要文字。 |
| F3 | 報名入口 | QR Code 下方標示行動感不足；可改成更明確的報名指引並保留掃描留白。 |
| F4 | 版面秩序 | 日期、地址、活動說明與聯絡資訊的左側起點和垂直間距可更一致。 |
| F5 | 視覺平衡 | 插畫與活動資訊之間留白偏大，底部文字集中；可調整上下比例與資訊區位置。 |
| F6 | 印刷對比 | 部分淺色文字印刷後可能不清楚；可加深顏色、提高字重並做紙本測試。 |

這些回饋必須都合理，但不能全部指向同一個修改；它們應促使參與者在 QR Code、資訊層級、文字精簡、對齊間距與視覺平衡之間做取捨。

### 材料 4: Text resources

至少提供：

- 5 個 main title / slogan。
- 3 個 subtitle。
- 2 到 3 種 description 長度。
- 必填資訊：date、time、location、CTA、QR caption、organizer。
- 允許參與者自創 slogan 或 rewrite text 時，需使用 `REWRITE [COMPONENT] -> "..."` 格式。

建議 ID：

- `S1` 到 `S5`: slogans。
- `ST1` 到 `ST3`: subtitles。
- `D1` 到 `D3`: description versions。
- `CTA1` 到 `CTA3`: call to action。

### 材料 5: Image / component library

提供 15 到 20 張圖片，並包括：

- 原海報既有圖片。
- 可替換主視覺圖片。
- 可作為背景或輔助圖像的圖片。
- 風格不同但都合理的選項。

建議 ID：

- `I01` 到 `I20`。

要求：

- 每張圖片要能用 ID 清楚引用。
- 圖片之間要有合理取捨，例如正式 vs 活潑、抽象 vs 具體、人物 vs 場景。
- 不要放明顯品質極差、所有人都會排除的選項。

### 材料 6: Visual elements and background

提供：

- 背景色或背景樣式：`BG1` 到 `BG5`。
- 裝飾元素：`E1` 到 `E8`。
- icon 或 emoji：`IC1` 到 `IC8`。
- layout hint 或 grid examples：`L1` 到 `L3`。

這些資源不一定要全被使用，目的是讓 action item 可以引用。

## 4. Component Taxonomy

正式 Canva material 需要替初始海報與 component library 標 component ID。以下 taxonomy 採用目前正式草案版本。

| Component ID | 中文名稱 | 說明 |
| --- | --- | --- |
| `C_TITLE` | 主標題 | 最大標題或活動名稱 |
| `C_SUBTITLE` | 副標題 | 活動短句、定位或補充說明 |
| `C_DESCRIPTION` | 說明文字 | 主要活動說明或段落 |
| `C_DATE` | 日期 | 日期資訊 |
| `C_TIME` | 時間 | 時間資訊 |
| `C_LOCATION` | 地點 | 地點或地址 |
| `C_CTA` | 行動呼籲 | 報名、掃碼、了解更多 |
| `C_QR` | QR code | QR code 圖像 |
| `C_QR_CAPTION` | QR 說明 | QR code 附近文字 |
| `C_HERO_IMAGE` | 主視覺 | 最大或最主要圖片 |
| `C_IMAGE_1` | 圖片 1 | 其他圖片 |
| `C_IMAGE_2` | 圖片 2 | 其他圖片 |
| `C_BACKGROUND` | 背景 | 背景圖或背景色 |
| `C_DECORATION` | 裝飾元素 | shape、icon、emoji、線條 |
| `C_ORGANIZER` | 主辦資訊 | 主辦、協辦、logo |
| `C_FOOTER` | 下方資訊 | footer 區域 |
| `C_SPACING` | 留白 / 間距 | 整體間距或區塊距離 |
| `C_ALIGNMENT` | 對齊 | 對齊方式或 grid |
| `C_TYPOGRAPHY` | 文字樣式 | 字型、大小、粗細、統一性 |
| `C_COLOR` | 色彩 | 色盤、對比、可讀性 |

## 5. Action Taxonomy

Action item 應使用下列 action 語彙；若系統可支援選單，優先使用這些作為選項。

| Action | 中文 | 用途 |
| --- | --- | --- |
| `ADD` | 新增 | 新增圖片、文字、CTA、icon、元素 |
| `REMOVE` | 去除 | 移除干擾或不必要元素 |
| `REPLACE` | 替換 | 用另一個 resource ID 替換 |
| `REWRITE` | 改寫 | 改寫文字內容 |
| `SHORTEN` | 縮短 | 縮短文字或資訊 |
| `CLARIFY` | 釐清 | 讓資訊更明確 |
| `REORDER` | 重排 | 改變資訊順序 |
| `MOVE` | 移動 | 改變位置 |
| `ALIGN` | 對齊 | 改變對齊方式 |
| `GROUP` | 分組 | 把相關資訊放在一起 |
| `SPLIT` | 拆分 | 拆開過密資訊 |
| `RESIZE` | 調整大小 | 通用尺寸調整 |
| `ENLARGE` | 放大 | 放大文字、圖片、區塊 |
| `SHRINK` | 縮小 | 縮小文字、圖片、區塊 |
| `EMPHASIZE` | 強調 | 提升視覺權重 |
| `DE_EMPHASIZE` | 弱化 | 降低視覺權重 |
| `CHANGE_COLOR` | 改顏色 | 改文字、背景或元素顏色 |
| `INCREASE_CONTRAST` | 提高對比 | 改善可讀性 |
| `CHANGE_FONT` | 調整字型 | 改變字型或字重 |
| `ADJUST_SPACING` | 調整間距 | 改善留白或密度 |
| `UNIFY` | 統一 | 統一字型、色彩、圖像風格 |
| `SIMPLIFY` | 簡化 | 減少資訊或裝飾 |
| `CROP` | 裁切 | 調整圖片裁切 |

## 6. Action Item 格式

建議系統內顯示欄位：

| 欄位 | 必填 | 說明 |
| --- | --- | --- |
| Component | 是 | 例如 `C_TITLE` |
| Action | 是 | 例如 `REPLACE` |
| Resource / Text | 視 action | 例如 `S3`、`I14` 或自創文字 |
| Short note | 否 | 簡短補充，不超過一句 |

建議顯示格式：

```text
[ACTION] [COMPONENT] -> [RESOURCE_OR_TEXT] ([SHORT_NOTE])
```

範例：

- `REPLACE C_TITLE -> S3`
- `ADD C_HERO_IMAGE -> I14`
- `REMOVE C_DECORATION -> E2`
- `MOVE C_QR -> C_FOOTER`
- `INCREASE_CONTRAST C_DESCRIPTION`
- `REWRITE C_TITLE -> "淨灘，讓海岸重新呼吸"`
- `REWRITE C_CTA -> "立即報名工作坊"`
- `UNIFY C_TYPOGRAPHY`

## 7. 系統資料

必須記錄：

- participant-created component + action items。
- private phase 1 think-aloud transcript。
- idea blocks。
- initial personal ranking。
- public discussion transcript。
- public chat messages。
- group ranking：最多 15 items。
- final personal ranking。
- cue logs：僅 with-cue 條件；no-cue 條件標記 `cue_enabled=false`。
- screen recording workaround：個別 Google Meet 錄影，不是 OmniObserve 系統功能。

## 8. 好任務材料的檢查標準

正式使用前，至少兩位研究團隊成員用 5 分鐘各自產生 action items，檢查：

- 是否自然出現 8 個以上不同 action items。
- 是否至少有 3 個設計方向彼此競爭。
- 是否有人會引用不同 slogans、images、background；若材料提供 reviewer feedback，也檢查是否有人會引用不同 feedback。
- 是否不需要直接編輯海報也能說清楚修改意圖。
- 是否不會全部集中在「主標題放大」「QR code 縮小」等單一明顯修正。
- 是否可在 5 分鐘內產生每人至少 4 個 component + action items。
- 是否所有 resource ID 清楚可讀。

## 9. 避免的材料設計

- 只有單一巨大錯誤，導致所有人提出同一件事。
- 可用圖片太少，讓討論變成二選一。
- 若材料提供 reviewer feedback，每個 feedback 都指向同一修正方向。
- 元件太多但沒有 ID，參與者無法引用。
- 要求參與者在任務中實際美編、拖拉或搜尋素材。
- 把所有圖片與素材塞進 OmniObserve UI，造成系統介面過載。
