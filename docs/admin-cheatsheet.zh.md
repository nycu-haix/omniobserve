# OmniObserve Admin Cheatsheet

本文件專供負責操作系統的 Admin 參考。Admin 的核心任務是 **切換實驗階段、控制提示（Cue）的發送條件、監控系統狀態與日誌（Logs）**，並確保實驗流程一致、觀察者不造成額外引導或偏誤。

## 本次實驗流程總覽

| 階段 | 時間 | 目標 |
| --- | ---: | --- |
| 階段 0：實驗準備與設備確認 | 前 10 分鐘 | 確認條件、設備、觀察者位置與後台狀態 |
| 階段 1：個人思考 / Private idea 產生 | 8 分鐘 | 完成個人初始排序，並至少留下 2 個 private thoughts / idea blocks |
| 階段 2：小組討論 / Public discussion | 20 分鐘 | 共同產生小組排序；每位參與者至少 public 發言 2 次 |
| 階段 3：討論後個人排序調整 | 2 分鐘 | 捕捉討論後的 final private ranking |
| 階段 4：資料確認與事後訪談 | 約 20 分鐘 | 確認 logs，追問講 / 不講、cue 影響與 critical incidents |
| 階段 5：Debrief 說明 | 約 10 分鐘 | 說明研究目的、提示機制與資料使用 |

---

## 階段 0：實驗準備與設備確認（前 10 分鐘）

### 要做什麼
1. 確認已經準備好 Admin 頁面（用於切換 phase、計時、發送 / 監控 Cue）。
2. 確認實驗條件是 **對照組**（無 Cue）還是 **實驗組**（有 Cue）。
3. 確保所有參與者的 Participant ID 分配正確，且各自的 Session Link 可以正常開啟。
4. 確認所有參與者都能正常操作：
   - 個人排序 / public ranking
   - private thoughts / Idea Box（若該條件啟用）
   - 麥克風靜音、公開發言、悄悄話 / private mode
   - 右側 transcript / chat / idea 區域
5. 確認 Admin board 能看到必要狀態：
   - 每位參與者是否在線
   - 每位參與者目前麥克風狀態
   - 是否正在 public speaking / private speaking / muted
   - Private thoughts、public transcript、cue logs 是否正常寫入
6. 準備好系統失效備案（例如螢幕截圖、畫面錄製或紙本紀錄）。

### 對照組 / 實驗組控制注意事項
- 若研究目標是比較 Cue effect，請盡量讓對照組與實驗組除了 Cue 之外的功能保持一致。
- 特別注意以下差異可能造成 confound：
  - 對照組是否有可輸入區域。
  - 對照組是否有 private typing / Idea Box。
  - 對照組是否能看到相同的 transcript / chat / ranking 操作介面。
- 若當天條件設計刻意保留差異，Admin 應在紀錄中明確標註，避免後續分析時直接把差異歸因於 Cue。

### 觀察者規則（避免 experiment bias）
- 觀察者不要坐在自己認識的受試者旁邊。
- 觀察者只回答 **technical issue**，例如「按鈕在哪裡」、「麥克風是否有開」、「頁面壞掉怎麼辦」。
- 觀察者不要主動鼓勵受試者發言，例如不要說「你可以講啊」、「有想法就說出來」、「你剛剛不是有寫嗎」。
- 觀察者不要主動提醒受試者使用某個策略，或暗示哪個物品該被討論。
- 觀察者不要長時間盯著受試者看；可假裝忙於記錄、看資料或做其他事，以降低受試者壓力。
- 若受試者講話過多且明顯偏離實驗流程，可用中性方式提醒回到規定任務，例如：「請記得你們需要在時間內完成共同排序。」

### 要注意什麼
- 若人手不足，Admin 可兼任其中一位觀察者（Participant Observer），但仍應遵守上述觀察者規則。
- 若 Admin board 無法看到所有人麥克風狀態，請記錄此問題，並請現場觀察者協助記錄受試者實際狀態。
- 若參與者混淆 private / public 區域，主持人只能重申系統規則，不要解讀其內容或引導其討論策略。

## 階段 1：個人思考 / Private idea 產生（8 分鐘）

### 說明講稿
> 現在請你自己思考 8 分鐘。
>
> 這個階段請先不要和其他人討論。
>
> 你需要完成兩件事：第一，排出你自己的物品順序；第二，請至少留下 2 個個人想法記錄，可以用文字輸入，也可以用語音記錄。
>
> 你的個人想法不需要很完整，可以是關鍵字、直覺、疑問、支持或反對某個物品的理由，或是你覺得某個物品排序太高或太低的原因。
>
> 這些內容主要是幫助你自己在等一下的小組討論中回想。系統不會自動把你的 private thoughts 公開給其他人，其他人也不會看到你的完整 private 內容。

### 要做什麼
1. 從 Admin 頁面將系統切換至「個人思考」階段並開始計時（8 分鐘）。
2. 監控參與者是否開始進行個人排序。
3. 監控後台是否正常收到參與者的 Private thoughts（文字輸入或語音轉文字 transcript）。
4. 確認每位參與者至少產生 2 個 private thoughts / idea blocks。
5. 若系統支援，確認 private thoughts 具有可被後續 Cue 使用的內容，例如：
   - 對某個物品的排序理由
   - 對某個物品重要性 / 不重要性的判斷
   - 對目前排序的疑問
   - 對某個排序可能太高 / 太低的想法

### 要注意什麼
- **[重要] 絕對不發送 Cue**：即使系統背景已計算出 Similarity pairs 或 Shared unspoken alignment，在這個階段也**不可以**發送任何 Cue 給參與者。
- 鼓勵參與者記錄排序原因，但不要要求他們講完整長篇理由。
- 若使用者只是留下很 general 的內容，例如「蚊帳很重要」但沒有任何原因，後續不一定適合觸發 Similarity Cue。
- 若受試者長時間停留在靜音狀態或似乎忘記切換 private mode，優先依靠系統提示；觀察者不要用口頭方式主動引導，除非受試者明確提出 technical issue。
- 若受試者擔心 private 內容是否被公開，可重申：「private thoughts 不會自動公開，是否帶進討論由你自己決定。」

## 階段 2：小組討論 / Public discussion（20 分鐘）

### 說明講稿

**對照組：**
> 現在請你們開始小組討論。
>
> 你們有 20 分鐘，需要在時間內產生一份小組共同排序。
>
> 請自由討論、提出理由、同意、反對、提問或修改排序。每位參與者請至少公開發言 2 次，發言可以很短，不需要一次講很長。最後請產生一份小組最終答案。

**實驗組：**
> 現在請你們開始小組討論。
>
> 你們有 20 分鐘，需要在時間內產生一份小組共同排序。
>
> 請自由討論、提出理由、同意、反對、提問或修改排序。每位參與者請至少公開發言 2 次，發言可以很短，不需要一次講很長。
>
> 系統有時候可能會根據你先前記錄的想法，提醒你有些內容可能和目前討論有關，或可能也有其他人有相近想法。你可以自行決定是否採納、忽略，或把它帶入討論。
>
> 系統不會自動替你公開你的個人想法，也不會公開其他人的個人想法。沒有看到提示也不代表沒有人和你想法相近，可能只是系統尚未判斷到合適時機。

### 要做什麼
1. 將系統階段切換至「小組討論」並開始計時（20 分鐘）。
2. 監控討論狀況，確認各參與者的麥克風狀態切換是否正常。
3. 監控是否每位參與者至少 public 發言 2 次。
4. 監控 public ranking 是否被移動、由誰移動、移動哪些 item，以及其他人是否注意到。
5. **[僅限實驗組] 發送 / 監控 Cue**：
   - 根據 Private thoughts、Public discourse 與參與者間的 Alignment，在適當時機發送 Cue。
   - Cue 的目標是幫助受試者意識到「自己私下記錄的想法可能和目前討論有關」，而不是替他公開內容。

### Cue 觸發條件
- **Shared Unspoken Alignment**：兩人以上私下表達相近的想法，但尚未公開。
- **Private-Public Mismatch**：參與者的個人想法與目前公開討論方向出現明顯落差。
- **Same Reason Similarity**：多位參與者針對同一物品有相似結論，且理由也相近。
- **Different Reason Similarity**：多位參與者針對同一物品有相似結論，但理由不同，可促進更完整討論。

### Cue 不應觸發的情況
- 只有單一 private thought，且沒有 Alignment、Mismatch 或與當前 public discussion 的明顯關聯。
- 內容過於 general，沒有具體理由、疑問、排序判斷或支持 / 反對依據。
- 目前討論已經離開該物品太久，Cue 可能延遲到失去脈絡。
- 短時間內已經對同一位參與者發送過多提示。

### Cue 呈現與數量原則
- **一次最多顯示 2 個 Cue**；若系統設定更保守，可一次只顯示 1 個。
- 避免 Public discussion 一開始一次跳出大量 Cue。
- 若同時存在多個 similar idea blocks，先從 strong candidates 中挑 1 個顯示；下一次再顯示其他 Cue。
- Cue 之間應有時間間隔，避免通知堆疊或讓參與者無法判斷優先順序。
- Cue 應短、少、可忽略；目標是 awareness，不是命令參與者發言。
- 若 Cue 會自動消失，Admin 應注意是否消失太快，導致受試者尚未處理就失去提示。

### Cue 顏色與類型
- **Same Reason：綠色**。
- **Different Reason：黃色**。
- 若系統支援，Cue 應以底色或 block 顏色呈現，而不只是改文字顏色。
- 若使用深淺表示強度，請確認深淺含義在研究紀錄中明確：
  - similarity score 較高
  - specificity 較高
  - 或目前優先級較高
- 若使用 similarity score 做深淺呈現，不建議固定用 `0.7–1.0` normalize；應根據目前 candidate cue 的最高分與最低分進行 normalize。

### Cue 優先順序參考
若同時有多個可提示內容，可優先考慮：
1. 目前 public discussion 正在討論的 item。
2. 與目前 public item / ranking 操作最相關的 private idea。
3. similarity score 較高的 pair。
4. specificity 較高、理由較具體的 idea。
5. 能補足目前討論缺口的 different reason。
6. 最近產生或最近被討論脈絡帶到的 idea（recency）。

### 提示文字參考
- 「你不是唯一對『{物品}』有這個方向想法的人。若你覺得合適，可以把你的觀點帶入討論。」
- 「你先前提到的『{想法摘要}』和目前討論可能有關。若你想補充，可以選擇現在或稍後提出。」
- 「目前有不只一個人私下提到與『{物品}』相關的疑問或支持理由。你可以決定是否提出自己的版本。」
- 「有人對『{物品}』有相近結論，但理由可能不同。若你覺得合適，可以補充你的理由。」

### 要注意什麼
- 不要把 Cue 當成「要求受試者發言」的指令。
- 不要讓受試者誤以為沒有 Cue 就代表沒有其他人有相近想法。
- 注意受試者是否不知道何時切入，即使看到 Cue 也可能仍然沒有發言。
- 注意 Cue 是否造成打斷、困惑、壓力或資訊過載。
- 注意 public ranking 的移動是否缺乏可見提示，導致其他人不知道是誰在移動。

---

## 階段 3：討論後個人排序調整（2 分鐘）

### 說明講稿
> 小組討論時間結束。接下來請先不要再公開討論。
>
> 現在請你回到自己的個人排序，你有 2 分鐘可以根據剛剛的討論調整自己的 private ranking。
>
> 這份排序代表你在小組討論後的個人最終想法。你可以調整，也可以維持原本排序。

### 要做什麼
1. 將系統切換至「討論後個人排序調整」階段並開始計時（2 分鐘）。
2. 確認參與者無法或不再進行公開討論。
3. 監控每位參與者是否完成 final private ranking。
4. 記錄討論後個人排序調整前後的差異。

### 要注意什麼
- 這個階段是為了捕捉討論後個人觀點是否改變，不是繼續小組協商。
- 不要要求參與者解釋為什麼改，解釋留到後測訪談再問。

---

## 階段 4：資料確認與事後訪談（約 20 分鐘）

### 討論結束資料確認講稿
> 時間到，排序調整到此結束。接下來我們會確認資料是否有正常記錄，並進行簡短訪談。

### 要做什麼：資料確認
1. 截取或從後台記錄當下介面上的**小組最終排序狀態**。
2. 確認每位參與者的 **final private ranking** 已正常記錄。
3. 確認各項日誌（Logs）皆有正常儲存：
   - 個人初始排序
   - 討論後 final private ranking
   - 小組最終排序
   - Private transcripts & idea blocks
   - 每位參與者 private idea block 數量
   - Group discussion transcript
   - 每位參與者 public 發言次數
   - Public ranking 操作紀錄（誰移動、何時移動、移動哪些 item）
   - 麥克風狀態與模式切換紀錄（public / private / muted）
   - Cue logs（僅限實驗組）
4. Cue logs 建議包含：
   - Cue 出現時間
   - 接收者
   - Cue 類型（Same Reason / Different Reason / Mismatch / Shared Unspoken Alignment）
   - Cue 顏色與呈現形式
   - 連結的 private idea blocks
   - 當下 public discussion 正在討論的 item
   - similarity score / specificity score（若有）
   - Cue 是否被看見、忽略、消失、或促成發言

### 訪談說明講稿
> 現在請你花幾分鐘時間接受訪談。我會根據你在討論中的一些行為和發言問你一些問題，幫助我們理解你是如何思考與決策的。
>
> 沒有標準答案，我們主要想了解你當下怎麼判斷要不要說、什麼時候說，以及系統提示或 private channel 是否有影響你。

### 訪談必問方向
1. 你剛剛有沒有任何 private idea 最後沒有拿出來公開講？為什麼？
2. 當時其他人在討論什麼？你是因為內容不確定、沒有信心，還是不知道什麼時候切入？
3. 有沒有任何 private idea 後來被你帶進公開討論？你為什麼決定說出來？
4. 你比較傾向用語音還是打字記錄 private thoughts？為什麼？
5. Private channel / Idea Box 對你來說比較像筆記工具、發言前緩衝，還是額外負擔？
6. 你有沒有擔心 private 內容被公開，或混淆 private / public 顯示區域？
7. 你有沒有注意到誰在移動 public ranking？這會不會影響你討論？
8. 排序差異標示、紅綠提示或 Similarity Cue 的顏色是否容易理解？

### 實驗組加問方向
1. 你有注意到系統提示嗎？
2. 哪一個提示讓你更想說、比較不想說，或沒有影響？
3. 你有沒有忽略某個提示？為什麼？
4. 提示出現的時間點是剛好、太早、太晚，還是打斷討論？
5. 如果沒有看到提示，你會不會解讀成「沒有人和我想法相近」？
6. Same Reason / Different Reason 的顏色或差異是否清楚？

### 要注意什麼
- 訪談應在安靜的環境進行，避免其他參與者聽到。
- 引導式問題應基於實際觀察到的行為，而非假設。
- 特別追問「為什麼講 / 不講」、「當時其他人在講什麼」、「是否不知道何時切入」。
- 若受試者提到 UI/UX 混淆，請記錄具體畫面與情境，不要只記錄一般抱怨。

---

## 階段 5：Debrief 說明（10 分鐘）

### 一般 Debrief 說明稿
> 謝謝你完成今天的研究。
>
> 我們這次研究主要想了解，在小組討論中，系統是否能幫助參與者把個人階段想到、但可能沒有說出來的想法帶入公開討論。特別是當不只一個人私下有相近的疑問、支持或不同意見時，系統能否提供適當提醒，讓參與者更有信心決定是否提出。
>
> 有些組別會看到系統提示，有些組別不會。這是我們研究設計的一部分，用來比較提示是否影響討論行為。
>
> 系統不會自動公開你的個人想法，也不會把你的 private reasoning 直接給其他人看。提示的目的是提供可能的 shared alignment 或討論時機，但是否提出、何時提出、怎麼提出仍由你自己決定。
>
> 你的資料會匿名處理，只用於研究分析。

### 若使用暗樁，需追加說明
> 另外，今天討論中有一位參與者是研究團隊安排的成員。他的任務是在討論中提出某些具有說服力、但不一定正確的觀點。
>
> 這是為了觀察當小組中出現較強勢的意見時，其他參與者是否會先在 private channel 留下不同想法，以及系統提示是否能幫助 shared unspoken disagreement 被帶入公開討論。
>
> 我們不會用這個設計評價你的能力或表現，而是分析系統與討論過程。
>
> 如果你對這個安排感到不舒服，可以告訴我們，我們可以刪除你的資料。

### 要做什麼
1. 主持人親自朗讀 debrief 說明稿給所有參與者。
2. 若實驗組使用了暗樁，**必須追加說明**暗樁安排。
3. 給予參與者提問機會。
4. 若參與者對暗樁安排表示不滿，告知其可以選擇撤回資料。

### 要注意什麼
- 所有參與者完成訪談後再進行 debrief（確保暗樁身分不被提前洩露）。
- Debrief 應在所有參與者同時進行，以保持公平透明。
- 若有使用暗樁但參與者表示反對，務必尊重其撤回資料的決定。

## Admin Troubleshooting Checklist

### 麥克風 / 模式切換
- 參與者是否長時間處於 muted？
- 參與者是否想使用 private channel，但忘記從 muted 切換到 private mode？
- Admin board 是否能清楚看到每位參與者目前是 public speaking、private speaking 還是 muted？
- 受試者是否擔心自己切錯模式，導致 private 內容被公開？

### Private / Public 邊界
- 受試者是否把右側 transcript 誤解為公開區？
- 受試者是否擔心 private thoughts 被其他人看到？
- 受試者是否希望 private transcript 與 public transcript 分開？

### Cue
- Public 一開始是否出現過多 Cue？
- Cue 是否一次超過 2 個？
- Cue 是否太快消失？
- Cue 是否延遲到原話題已經過去？
- Cue 是否造成受試者困惑、停頓、焦慮或打斷？
- Same Reason / Different Reason 顏色是否正常？

### Public ranking
- 其他人是否知道誰正在移動排序？
- 排序移動是否太不明顯，導致討論和操作脫節？
- 是否有 item 突然移動但沒有人知道原因？

### 資料紀錄
- 個人初始排序、final private ranking、小組最終排序是否都有存到？
- Private thoughts / idea blocks 是否都有 participant ID 與 timestamp？
- Public transcript 是否完整？
- Cue logs 是否包含 cue 類型、接收者、時間與連結 idea？
- 實驗結束後是否已匯出 Logs 備份？

### 任務說明文字
> 你們搭乘一艘私人遊艇，在南太平洋上漂流。因為一場原因不明的火災，遊艇和船上大部分物品都被燒毀，遊艇正在慢慢下沉。
>
> 由於重要的導航設備被破壞，而且大家在滅火時分心，你們無法確定目前位置。你們目前最好的估計是：距離最近的陸地約 1,600 公里，而且大約位在該陸地的南南西方。
>
> 在遊艇完全沉沒前，你們成功救出一艘可用的橡皮救生艇和船槳，救生艇足以載下所有人和下列 15 項物品。所有人生還者口袋裡的東西合計只有：一包香菸、幾盒火柴，以及五張 1 美元紙鈔。
>
> 下列 15 項物品都沒有受損，可以正常使用。這些物品不能連上網路，也不能直接呼叫救援；請根據它們在海上求生時的實際用途排序。
>
> 你們的任務是：**依照這些物品對「海上求生」的重要性進行排序。**
>
> 你們會先各自思考 8 分鐘，產生自己的排序與理由。這個階段請不要和其他人討論，並請至少留下 2 個個人想法記錄。
>
> 接著你們會進入 20 分鐘的小組討論。你們需要一起討論並產生一份小組最終排序，每位參與者請至少公開發言 2 次。
>
> 小組討論後，你們會有 2 分鐘可以調整自己的個人最終排序。
>
> 請注意，過程中不要上網查答案，也不要使用任何外部工具搜尋這個任務。這個任務的重點不是考你是否知道標準答案，而是觀察你們如何討論、表達理由、形成共識。

### 專家解答（海岸防衛隊標準答案）

求生最高指導原則：「吸引救援（發出訊號）」比「維持生命」更重要，而「導航」在沒有其他設備的情況下毫無用處。

| 排名 | 物品 | 專家解答與原因 |
| :---: | :--- | :--- |
| **1** | **刮鬍鏡／小鏡子** | **（極重要：發出訊號）**<br>在海面上最強大的視覺求救工具。太陽光的反射可以被遠在幾英里外的飛機或船隻看見。 |
| **2** | **8L 油氣混合燃料** | **（極重要：發出訊號）**<br>可以倒在水面上點燃，它會漂浮在海面上燃燒，是極佳的求救訊號。 |
| **3** | **20L 飲用水** | **（重要：維持生命）**<br>補充水分是生存的基礎，比食物更為重要。 |
| **4** | **一箱緊急口糧** | **（重要：維持生命）**<br>提供基本生存所需的熱量與營養。 |
| **5** | **2m² 不透明塑膠布** | **（實用：收集資源／遮蔽）**<br>可以用來收集雨水，或是作為遮陽避雨的掩蔽物。 |
| **6** | **兩盒巧克力棒** | **（實用：維持生命）**<br>提供快速補充體力的能量來源。 |
| **7** | **釣魚工具組** | **（次要：維持生命）**<br>雖然可以提供食物來源，但捕獲量不穩定，且生吃海鮮可能會導致口渴。 |
| **8** | **5m 尼龍繩** | **（次要：工具）**<br>可以用來將設備或人員綁在一起，防止在海浪中失散。 |
| **9** | **救生坐墊** | **（次要：救生）**<br>萬一有人落水，可以作為漂浮物救援使用。 |
| **10** | **防鯊劑** | **（心理作用大於實質）**<br>雖然有一定作用，但主要是提供心理上的安慰，避免恐慌。 |
| **11** | **1L 80% 蘭姆酒** | **（有害：加速脫水）**<br>雖然含有高濃度酒精可以用來消毒外傷，但絕不可飲用，因為酒精會嚴重加速身體脫水。 |
| **12** | **電晶體收音機** | **（幾乎無用）**<br>在茫茫大海中通常收不到訊號，無法發揮作用。 |
| **13** | **太平洋地圖** | **（無用）**<br>沒有精確的導航儀器，光有地圖根本無法知道自己在哪裡。 |
| **14** | **蚊帳** | **（無用）**<br>太平洋中央沒有蚊子。 |
| **15** | **六分儀** | **（完全無用）**<br>六分儀需要配合精密的天文鐘和星曆表才能測量經緯度，單獨存在毫無用處。 |
